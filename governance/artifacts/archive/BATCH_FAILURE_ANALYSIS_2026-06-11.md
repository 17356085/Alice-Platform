## 2. 失败分类与模式识别

### 类型 1：渲染时序（~40 例，30%）

| 特征 | 描述 |
|------|------|
| 受影响页面 | user, customer, notice, login_log, operation_log, org, system_log, timed_task |
| 失败模式 | `is_page_loaded()` 返回 True，但后续 `get_table_*` 方法获取到空数据 |
| 根因 | 页面级 `is_page_loaded` 只检查表格容器存在，不等待具体内容渲染 |
| 修复方案 | 在各 PageObject 的 `is_page_loaded()` 中增加内容就绪检测 |
| 预估工时 | 1h → +25 通过 |

### 类型 2：表头断言（~30 例，23%）

| 特征 | 描述 |
|------|------|
| 受影响页面 | alarm_config, sensor, unit, maintenance, employee, post, question_bank, paper, exam, sales_order |
| 失败模式 | `get_table_header_texts()` 返回空或部分列头 |
| 根因 | Vue v-for 异步渲染 `<th>` 内容，表格容器先于列头文本渲染 |
| 修复方案 | 全局 `BasePage.get_table_headers()` JS提取+6次重试+稳定检测（报告 § P2 已修复，部分页面待确认） |
| 预估工时 | 0.5h → +20 通过 |

### 类型 3：CRUD 弹窗（~35 例，26%）

| 特征 | 描述 |
|------|------|
| 受影响页面 | contract(10F), role(6F), user(6F), train_plan(9F) |
| 失败模式 | 弹窗定位器找不到按钮/表单字段 |
| 根因 | contract 是 API 脱节（Page Object API 重构后测试未更新），其余为定位器差异 |
| 修复方案 | contract 重写对应测试；role/user/train_plan 修复定位器 |
| 预估工时 | 2h → +20 通过 |

### 类型 4：搜索/筛选（~15 例，11%）

| 特征 | 描述 |
|------|------|
| 受影响页面 | menu(4F filter), sensor(3F search), notice, operation_log, org, unit |
| 失败模式 | 搜索/筛选后结果断言失败 |
| 根因 | 搜索后等待不充分、筛选条件定位器变化 |
| 修复方案 | 增加搜索后 `wait_loading_disappear()` + 特定页面的筛选定位器修正 |
| 预估工时 | 1h → +10 通过 |

### 类型 5：RBAC 权限（12 例，9%）

| 特征 | 描述 |
|------|------|
| 受影响页面 | test_rbac_permission / test_rbac_check |
| 失败模式 | 权限校验失败——测试需要的角色/用户不存在 |
| 根因 | **种子数据缺失**——RBAC 测试所需的特定角色和权限未在 test 环境中创建 |
| 修复方案 | 运行 `script/system/rbac_seed_final.py` 创建种子数据 |
| 预估工时 | 0.5h → +12 通过 |

## 4. 预防措施

1. **CI 前置检查**：每次构建前运行种子数据脚本，确保环境数据一致
2. **is_page_loaded() 规范**：全局约定"页面加载完成 = 表格容器存在 + 至少1行数据 或 空数据提示出现"
3. **定位器健康监控**：提取所有 A/B 级定位器到配置表，批量验证可用性
4. **渲染等待基线**：`page_load_timeout` 保持 60s，`table_render` 增加 8s→12s
