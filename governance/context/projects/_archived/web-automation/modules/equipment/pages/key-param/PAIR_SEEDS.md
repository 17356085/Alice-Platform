# PAIR_SEEDS — 关键参数监控

> 结对同学：AI Pilot Test | 日期：2026-06-22 | 需求ID：TAPD-EQ-PARAM-001

## 种子场景

### SEED-001：参数阈值越界触发报警联动

- **优先级**: P0
- **类型**: positive
- **业务背景**: 关键参数（温度/压力/液位）超过阈值后应自动触发报警工单。AI 从页面分析只能看到"参数展示"和"阈值配置"，不知道报警联动这个跨模块业务流程。这是生产安全的核心闭环。
- **前置条件**: 1. 已配置温度参数阈值上限 80°C 2. 模拟数据源推送温度值 85°C
- **操作步骤（概要）**:
  1. 打开关键参数监控页面
  2. 观察温度参数的显示值和颜色
  3. 切换到报警管理页面（alarm-config）
  4. 检查是否自动生成了报警记录
- **预期结果**: 1. 关键参数页面温度值显示为红色（超阈值）2. 报警管理页面自动生成一条"温度过高"报警 3. 报警级别为"紧急"
- **关键检查点**: 跨页面联动——参数监控页面的视觉告警 → 报警配置页面的记录生成，需要端到端验证。

### SEED-002：历史数据曲线的时间范围切换

- **优先级**: P1
- **类型**: boundary
- **业务背景**: 历史数据曲线默认显示最近 1 小时。运维人员排查问题时经常需要切换到"最近 24 小时"或自定义范围。AI 容易忽略图表渲染——切换时间范围后图表是否正确重绘，数据点是否连续。
- **前置条件**: 系统中已积累 ≥24 小时的参数历史数据
- **操作步骤（概要）**:
  1. 打开关键参数监控页面
  2. 点击"历史曲线" tab
  3. 默认显示最近 1 小时 → 确认图表有数据
  4. 切换时间范围为"最近 24 小时"
  5. 切换为自定义范围（选择 7 天前到今天）
- **预期结果**: 1. 每次切换后图表重新渲染，数据点数量与时间范围匹配 2. 自定义范围灵活可用 3. 无数据时应显示"暂无数据"而非空白图表
- **关键检查点**: 图表使用 ECharts Canvas——Selenium 不能直接验证数据点，需用 JS 检查 `echarts.getInstanceByDom()` 的 option。

### SEED-003：多参数同时实时更新不卡顿

- **优先级**: P1
- **类型**: boundary
- **业务背景**: 页面可能同时展示 10+ 个实时参数，每个参数每秒更新一次。AI 通常测单个参数的显示正确性，忽略"大量数据高频更新"时的页面性能——可能导致浏览器卡顿、数据延迟、甚至崩溃。
- **前置条件**: 1. 已配置 ≥10 个实时参数 2. 数据源以 1Hz 频率推送
- **操作步骤（概要）**:
  1. 打开关键参数监控页面
  2. 保持页面打开 3 分钟
  3. 观察页面是否卡顿
  4. 在 3 分钟内在不同 tab（实时/历史/报警）之间切换
- **预期结果**: 1. 3 分钟内页面不卡顿、不崩溃 2. 参数值持续更新（最后更新时间戳不早于 2 秒前） 3. tab 切换流畅
- **关键检查点**: 内存泄漏——Selenium 可通过 `performance.getEntries()` 检查 JS heap size 是否持续增长。

## 补充说明

- **特殊业务规则**: 不同参数类型的阈值范围不同——温度（-50~200°C）、压力（0~25MPa）、液位（0~罐容的 95%）。配置阈值时必须校验范围合法性。
- **已知坑点**: 实时数据通过 WebSocket 推送，Selenium 的 `page_source` 是快照，看不到实时更新。验证实时性需用 `execute_script` 读取 DOM 内容或 WebSocket 消息计数。
- **不需要测试的场景**: 图表的美观度（颜色/线宽/标注），数据采集精度（需要硬件传感器配合）。
