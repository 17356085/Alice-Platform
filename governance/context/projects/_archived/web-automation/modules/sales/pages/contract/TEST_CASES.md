#### 自动化优先级建议

**高优先级（P0，核心安全与功能）**：
1.  **核心增删改查**：创建、编辑、删除、搜索的 Happy Path 和必填项校验。
2.  **状态机流转**：关键操作（如提交审批、审批通过）后的状态变更。
3.  **权限控制**：数据隔离、URL直接访问、按钮显隐。
4.  **Token过期处理**：这是安全红线，必须自动化。

**中优先级（P1，重要体验与数据完整性）**：
1.  **并发冲突**：模拟多人同时编辑，验证乐观锁机制。
2.  **组合搜索与筛选**：验证多维度的数据过滤。
3.  **异常网络接口响应**：超时、500错误、异常数据格式。
4.  **审批流流转**：验证角色的完整旅程（从提交到签署）。

**低优先级或不宜自动化（P2，体验与探索）**：
1.  **不建议自动化**：
    *   **多浏览器兼容性测试**（如富文本编辑器在不同浏览器下的布局）。
    *   **UI视觉效果**（如颜色的精确对比，建议探索性测试）。
    *   **长时间的大数据量性能测试**（应使用专门的性能测试工具）。
2.  **可自动化但价值低**：
    *   简单的列宽调整、排序操作（除非对功能有核心影响，否则手工更快）。
    *   特殊字符搜索的极少数边缘案例。
