| TC-009 | 删除审批链 - 确认弹窗中取消 | P2 | ✅ | 边界操作。确认弹窗(`confirm`)定位器稳定。 |

**风险标注:**
- TC-004(`el-switch`)：定位器为B+级，其状态变更（选中/取消）可能因页面动画或异步请求而延迟，需添加显式等待等待状态类(`is-checked`)变更。
- TC-005(`el-select`)：定位器为B级，其下拉弹出层元素是动态生成并挂载到`<body>`上的，需使用Selenium的`move_to_element`或`ActionsChains`选择，增加脚本复杂度。

### 2. PageObject 拆分方案

基于“一个页面一个Page类，复杂弹窗独立”原则，建议如下拆分：

```
pages/
├── __init__.py
├── module_workflow/
│   ├── __init__.py
│   ├── approval_chain_page.py          # 主Page类
│   └── approval_chain_dialog.py        # 审批链弹窗Page类
│   └── approval_chain_detail_page.py   # 右侧详情区域Page类
```

- **`ApprovalChainPage`：** 主页面对象，负责左侧面板的操作。
    - 职责：定位树组件、执行新增/搜索/删除等操作。
    - 方法：`expand_node()`, `click_add_button()`, `search_chain()`, `select_chain()`等。
- **`ApprovalChainDetailPage`：** 右侧详情区域，负责编辑审批链内容。
    - 职责：修改名称、启停开关、管理审批节点。
    - 方法：`fill_name()`, `set_switch()`, `click_save()`等。
- **`ApprovalChainDialog`：** 独立弹窗，负责新增或编辑审批链时出现的弹窗。
    - 职责：包含名称输入、节点配置等。
    - 方法：`fill_name()`, `add_approver()`, `click_confirm()`等。

### 3. 公共组件复用分析

- **可直接复用 BasePage 方法：**
    - `wait_for_element_visible()` — 等待页面元素可见。
    - `click_element()` — 点击新增/保存/编辑等按钮。
    - `input_text()` — 填写审批链名称、搜索框。
    - `find_element_by_text()` — 在树组件中找到特定文本的节点。

- **需要扩展 ElementPlusHelper：**
    - `select_option()` — 新增一个方法处理`el-select`下拉选择，定位下拉层并选择选项。
    - `switch_check_status()` — 新增方法用于获取/设置`el-switch`的选中状态，并等待状态变更完成。

### 4. 等待策略建议

该页面存在以下异步行为：
- **树组件加载**：点击搜索或新增后，树组件需要异步刷新。建议使用`wait_for_element_by_text()`等待树中目标节点标签出现。
- **新增/编辑弹窗加载**：点击“新增”按钮后，等待弹窗DOM元素出现（例如`dialog.visible`为true），再执行后续操作。
- **保存成功后页面跳转/弹窗关闭**：点击“保存”后，等待“保存成功”的提示出现并消失，或者等待弹窗关闭按钮可见性改变。
- **启用/停用开关**：点击开关后，等待`el-switch__core`上的`is-checked` class改变。

**建议封装：**
- `def wait_for_tree_node_text(text: str, timeout=10):`
- `def wait_for_dialog_open(dialog_ele, timeout=10):`
- `def wait_for_switch_state_change(switch_ele, target_state: bool, timeout=5):`
