# BUSINESS_SCENARIOS — lab / water-compare

## 1. 业务目标

| 场景ID | 业务目标 | 描述 | 置信度 |
|--------|----------|------|--------|
| BS-WC-001 | 核心目标：对比两个不同取样点的水质分析结果 | 用户进入页面是为了快速查看两个采样位置在同一时间段内的水质指标差异，辅助分析水质变化趋势或设备运行状态。 | ✅ 已确认 |
| BS-WC-002 | 次要目标：验证特定时间段内水质数据是否异常 | 通过日期筛选查看历史对比，定位数据突变点。 | ✅ 已确认 |
| BS-WC-003 | 非目标：不进行水质数据的增删改操作 | 页面只读，不提供编辑、删除、导出功能。 | ✅ 已确认 |

## 2. 角色与旅程

| 场景ID | 角色 | 典型旅程 | 置信度 |
|--------|------|----------|--------|
| BS-WC-004 | 化验员 | 登录 → 导航到化验室取样 → 水质分析 → 水质分析对比 → 选择两个取样位置（如A1和B2）→ 设置日期范围（最近7天）→ 点击查询 → 查看对比表格 → 分析差异 → 关闭页面 | ✅ 已确认 |
| BS-WC-005 | 化验室主管 | 登录 → 导航到相同页面 → 选择不同部门的位置 → 设置更宽日期范围（如一个月）→ 查询 → 审视整体数据 → 导出（若有） | ⚠️ 推断 |
| BS-WC-006 | 无权限用户（如气化站操作员） | 通过书签直接访问 → 返回401或跳转到首页 → 无法看到页面内容 | ⚠️ 推断 |

## 3. 业务流程

| 场景ID | 流程类型 | 流程描述 | 关联页面 | 置信度 |
|--------|----------|----------|----------|--------|
| BS-WC-007 | Happy Path | 用户选择两个不同位置 + 有效日期范围 → 查询 → 表格展示对比数据（至少一行） → 用户可滚动查看全部水质指标 | water-compare | ✅ 已确认 |
| BS-WC-008 | Alternative Path 1 | 用户选择两个相同位置 → 查询 → 表格展示数据相同，无差异 → 用户应看到提示“请选择不同位置”或数据无差异的提示 | water-compare | ✅ 已确认 |
| BS-WC-009 | Alternative Path 2 | 用户选择的位置在所选日期内无数据 → 查询 → 表格显示空状态提示“当前时间范围无对比数据” | water-compare | ✅ 已确认 |

| BS-WC-011 | 异常流程 | 网络中断或后端故障 → 页面显示错误提示并保持表单可交互 | water-compare | ✅ 已确认 |

## 4. 业务规则

| 场景ID | 规则分类 | 具体规则描述 | 可验证性 | 置信度 |
|--------|----------|-------------|----------|--------|
| BS-WC-012 | 校验规则 | 选择的位置必须来自水质采样点表（系统配置），不能手动输入 | 验证下拉选项来源 | ✅ 已确认 |
