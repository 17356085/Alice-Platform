# TEST_DESIGN — personnel / entry-confirm

## 基本信息
- 模块：personnel
- 页面：入场确认
- 测试目标：验证入场确认列表展示、搜索筛选、单条确认入场、批量确认入场、分页
- 需求来源：PAGE_CONTEXT.md + 线上 DOM 诊断（2026-06-15）

## 测试范围
- 包含：页面加载、搜索（按承包商名称/人员姓名）、重置、分页、单条确认入场、批量确认入场
- 不包含：权限校验、清空缓存功能、并发确认

## 数据策略
- 测试数据来源：依赖线上入场审批通过后流转至此的记录
- 数据清理方式：确认入场操作不可逆，需谨慎操作
- 前置准备：确保存在至少2条"未读"（待确认）记录
- ⚠️ 批量确认会修改多条数据，测试后无法恢复

## 自动化建议
- 建议自动化：列表/搜索/P0确认流程 — 依赖现有测试数据
- 不建议自动化：并发确认、清空缓存
- 风险：如果线上无待确认数据，确认操作自动化将失败；批量确认会永久修改数据
