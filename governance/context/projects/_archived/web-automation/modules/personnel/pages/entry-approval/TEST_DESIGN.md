# TEST_DESIGN — personnel / entry-approval

## 基本信息
- 模块：personnel
- 页面：入场审批
- 测试目标：验证入场审批列表展示、搜索筛选、通过/驳回操作、详情查看及状态流转
- 需求来源：PAGE_CONTEXT.md + 线上 DOM 诊断（2026-06-15）

## 测试范围
- 包含：页面加载、搜索（按申请人/单位/状态）、分页、审批通过、审批驳回（含审批意见）、查看详情
- 不包含：权限校验、审批链流转（多级审批）、审批撤回

## 数据策略
- 测试数据来源：依赖线上/"承包商人员"发起的入场申请
- 数据清理方式：审批操作不可逆，不在自动化中恢复 — 使用已存在数据 + 仅执行只读/状态切换
- 前置准备：确保存在至少1条"待审批"记录

## 自动化建议
- 建议自动化：列表/搜索/P0审批流程 — 依赖现有测试数据
- 不建议自动化：并发审批、审批链多级流转
- 风险：如果线上无待审批数据，审批操作自动化将失败 → 需在测试数据准备阶段创建入场申请
