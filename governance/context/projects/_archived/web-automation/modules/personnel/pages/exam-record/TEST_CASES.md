## 自动化优先级建议

| 优先级 | 建议说明 | 对应用例 |
| :--- | :--- | :--- |
| **P0 (强制自动化)** | 核心流程、权限安全、删除确认，防止阻塞性缺陷 | TC-EXAM-LD-001, LD-003, DL-001, DL-002, PM-002, ERR-002, BS-001 |
| **P1 (推荐自动化)** | 主要功能、关键异常，减少手工回归成本 | TC-EXAM-LD-002, SR-001, SR-002, SR-003, TB-001, TB-002, PM-001, ERR-001, BS-002 |
| **P2 (可选或手动)** | 边缘场景、特殊字符、排序、跨角色隔离（数据准备复杂） | TC-EXAM-SR-004, TB-003(排序), BS-003(权限隔离) |

### 不建议自动化的场景

- **BS-003 (跨角色隔离验证)**：需要准备多套班组数据（班组B/班组C），且涉及不同角色的登录配置，数据依赖复杂，自动化成本高。建议作为手工探索性测试，或通过**后端API自动化**实现数据隔离校验，而非UI自动化。
- **ERR-004 (网络断开)**：此场景是浏览器原生行为（无连接提示），WebDriver无法模拟操作系统级别的网络断连，UI自动化无法可靠验证，应作为手工测试。
