# TEST_DESIGN — personnel / entry-record

## 基本信息
- 模块：personnel
- 页面：入场记录
- 测试目标：验证入场记录列表展示、多维度搜索（姓名/单位/日期范围）、导出、详情查看
- 需求来源：PAGE_CONTEXT.md + 线上 DOM 诊断（2026-06-15）

## 测试范围
- 包含：页面加载、搜索（按姓名/单位/日期范围）、分页、导出、详情查看、重置
- 不包含：新增/编辑/删除（只读页面）、数据统计

## 数据策略
- 测试数据来源：依赖入场审批通过后自动生成的记录
- 数据清理方式：只读页面，不产生清理需求
- 前置准备：无需特殊准备，0条数据也可验证页面加载

## 自动化建议
- 建议自动化：列表/搜索/详情/导出 — 100%自动化，只读无副作用
- 不建议自动化：无（纯只读页面）
- 特殊处理："共 0 条"的数据断言需容错
