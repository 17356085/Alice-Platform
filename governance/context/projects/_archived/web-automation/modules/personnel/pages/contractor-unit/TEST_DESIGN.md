# TEST_DESIGN — personnel / contractor-unit

## 基本信息
- 模块：personnel
- 页面：承包商单位
- 测试目标：验证承包商单位的 CRUD 全流程、搜索筛选、分页、启停用及异常处理
- 需求来源：PAGE_CONTEXT.md + 线上 DOM 诊断（2026-06-15）

## 测试范围
- 包含：页面加载、搜索（按名称/编码/状态）、分页、新增、编辑、启停用、删除、重置
- 不包含：权限校验（需专门权限测试账号）、并发操作、多用户协作

## 数据策略
- 测试数据来源：运行时 `datetime.now()` 生成唯一编码/名称（`CU{timestamp}` + `测试承包商_{timestamp}`）
- 数据清理方式：`conftest._teardown_contractor_unit()` 按名称前缀搜索删除
- 前置准备：admin 登录即可

## 自动化建议
- 建议自动化：搜索/CRUD/分页/重置 — 定位器基于标准 Element Plus 组件，A级占比>70%
- 不建议自动化：重复编码校验（需要事先有特定数据）
- 自动率目标：P0 100%，P1 100%，P2 按需
