# TEST_DESIGN — system / timed-task

## 基本信息
- 模块：system
- 页面：定时任务
- 测试目标：验证任务CRUD、状态筛选、Cron生成器、执行操作
- 需求来源：PAGE_CONTEXT.md + RISK_MODEL.md

## 测试范围
- 包含：页面加载、状态筛选(全部/运行中/已暂停)、搜索、新增/编辑/删除任务、分页
- 不包含：任务实际执行结果验证（需后端环境配合）、Cron可视化生成器UI验证

## 数据策略
- 测试数据来源：运行时生成唯一任务名称(时间戳后缀)
- 数据清理方式：fixture teardown中按名称前缀删除
- 前置准备：admin账号登录

## 自动化建议
- 建议自动化：搜索/状态筛选/CRUD — 定位器稳定
- 不建议自动化：Cron可视化生成器（抽屉组件复杂）
- 原因：Cron生成器使用第三方组件，DOM结构不稳定
