# TEST_DESIGN — system / login-log

## 基本信息
- 模块：system
- 页面：登录日志
- 测试目标：验证页面加载、搜索筛选、详情查看、清空、导出（只读审计页面）
- 需求来源：PAGE_CONTEXT.md + RISK_MODEL.md

## 测试范围
- 包含：页面加载、搜索(用户名/状态/时间)、详情弹窗、清空(确认/取消)、分页、导出
- 不包含：大数据量性能测试、跨时区时间准确性测试

| 场景ID | 测试维度 | 场景描述 | 优先级 | 执行方式 |
|--------|----------|----------|--------|----------|
| TD-LOGINLOG-001 | 页面加载 | 登录日志页正常加载，表格显示 | P0 | 自动化 |

| TD-LOGINLOG-012 | 搜索 | 按登录时间范围搜索 | P1 | 自动化 |
| TD-LOGINLOG-013 | 搜索 | 重置搜索条件 | P1 | 自动化 |
| TD-LOGINLOG-020 | 详情 | 查看登录日志详情弹窗 | P1 | 自动化 |

## 数据策略
- 测试数据来源：依赖系统已有登录记录（无需造数据）
- 数据清理方式：不清理（只读页面）
- 前置准备：admin账号登录（确保有登录日志数据）

## 自动化建议
- 建议自动化：页面加载/搜索/详情/分页 — 定位器稳定
- 不建议自动化：导出文件内容验证、大量数据搜索性能
