# AUTO_STRATEGY — system / timed-task

## 自动化目标
- 覆盖定时任务页 P0 全量 + P1 ≥80%，预计 11 条自动化用例

## 推荐策略
- 脚本层级：TimedTaskPage → conftest.py → test_timed_task_management.py
- 断言层级：页面级 + 数据级 + 流程级(CRUD)
- 数据策略：运行时生成唯一任务名称(时间戳后缀)，teardown中清理
- 清理策略：teardown中按名称前缀删除

## 适合自动化的场景
- 状态筛选 — radio-button定位器A级
- 搜索/CRUD — 表单字段固定
- 分页 — 标准el-pagination

## 不适合自动化的场景
- Cron可视化生成器 — 第三方组件DOM不稳定
- 任务执行/暂停/恢复 — 需后端调度环境
- 任务日志查看 — 独立页面/弹窗

## 依赖条件
- 环境：test环境，admin账号
- 权限：仅admin可访问
