# AUTO_STRATEGY — system / operation-log

## 自动化目标
- 覆盖操作日志页 P0 全量 + P1 ≥80%，预计 10 条自动化用例（只读页面）

## 推荐策略
- 脚本层级：OperationLogPage → conftest.py → test_operation_log_management.py
- 断言层级：页面级 + 交互级(弹窗/筛选) + 数据级(搜索结果)
- 数据策略：依赖系统已有操作记录，无需造数据
- 清理策略：不清理（只读页面）

## 适合自动化的场景
- 多条件搜索 — 各输入框定位器A级
- 状态筛选 — radio-button稳定
- 清空弹窗交互 — 仅验证弹窗+取消

## 不适合自动化的场景
- 时间范围搜索边界验证 — 数据依赖
- 详情弹窗内容完整性 — 含动态JSON数据
- 敏感信息脱敏验证 — 需人工检查

## 依赖条件
- 环境：test环境，admin账号
- 数据：系统至少有一些操作记录
- 权限：仅admin可访问
