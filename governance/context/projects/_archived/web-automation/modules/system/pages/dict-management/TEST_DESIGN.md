# TEST_DESIGN — system / dict-management

## 基本信息
- 模块：system
- 页面：字典管理
- 测试目标：验证字典分类切换、字典明细CRUD、搜索筛选全流程
- 需求来源：PAGE_CONTEXT.md + RISK_MODEL.md

## 测试范围
- 包含：页面加载、分类切换/筛选/搜索、字典CRUD、搜索、分页
- 不包含：字典值被业务引用的集成测试

## 数据策略
- 测试数据来源：运行时生成唯一标签(时间戳后缀)
- 数据清理方式：闭环删除（新增→搜索验证→删除→确认无）
- 前置准备：admin账号登录，选择"自定义"分类

## 自动化建议
- 建议自动化：分类切换/搜索/CRUD — 定位器稳定
- 不建议自动化：大数据量分类列表性能
