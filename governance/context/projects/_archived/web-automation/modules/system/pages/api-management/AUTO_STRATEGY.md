## 4. 等待策略建议

该页面主要的异步行为来自与后端 API 的交互。

| 操作 | 异步行为 | 建议等待策略 |
| :--- | :--- | :--- |
| 点击“搜索”/“重置” | 触发新的列表数据请求 | 等待表格内 `v-loading` 指令移除，或等待表格某行数据（如第一行）发生变化。 |
| 点击“新建”/“编辑” | 请求后端获取下拉选项（如请求方法预设值），加载弹窗内容 | 等待 `el-dialog` 的 `dialog-visible` 属性为 true，且弹窗内表单选项加载完成。 |
| 点击“确定”提交表单 | 向后端发送保存请求，成功后关闭弹窗并刷新表格 | 等待 `el-dialog` 关闭，然后等待表格重新加载（同搜索操作）。 |
| 点击“删除”确认删除 | 向后端发送删除请求，成功后刷新当前页表格 | 同上。 |
| 切换分页 | 请求新页数据 | 等待表格 `v-loading` 指令移除。 |

**建议封装**：
- 在 `BasePage` 或工具类中，提供 `wait_for_loading_spinner()` 方法，用于等待任何 `element-loading` 属性控制的加载状态结束。
- 提供一个更具体的 `wait_for_table_load()` 方法，它调用 `wait_for_loading_spinner()`，然后等待表格主体出现。
