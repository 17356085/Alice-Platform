# TEST_DESIGN — system / notice-management

## 基本信息
- 模块：system
- 页面：通知管理
- 测试目标：验证通知CRUD全流程、搜索筛选、富文本编辑
- 需求来源：PAGE_CONTEXT.md + RISK_MODEL.md

## 测试范围
- 包含：页面加载、搜索(标题/类型)、新增/编辑/删除通知、分页
- 不包含：富文本编辑器XSS安全测试（手工安全测试）、首页通知栏集成测试

## 场景设计
| 场景ID | 测试维度 | 场景描述 | 优先级 | 执行方式 |
|--------|----------|----------|--------|----------|
| TD-NOTICE-001 | 页面加载 | 通知管理页正常加载 | P0 | 自动化 |
| TD-NOTICE-010 | 搜索 | 按通知标题搜索 | P0 | 自动化 |
| TD-NOTICE-011 | 搜索 | 按通知类型下拉筛选 | P1 | 自动化 |
| TD-NOTICE-012 | 搜索 | 重置搜索条件 | P1 | 自动化 |
| TD-NOTICE-020 | 新增 | 新增通知-合法数据(含富文本内容) | P0 | 自动化 |
| TD-NOTICE-021 | 新增 | 标题为空提交 | P1 | 自动化 |
| TD-NOTICE-030 | 编辑 | 编辑通知内容 | P0 | 自动化 |
| TD-NOTICE-040 | 删除 | 删除通知 | P0 | 自动化 |
| TD-NOTICE-041 | 删除 | 取消删除 | P1 | 自动化 |
| TD-NOTICE-050 | 分页 | 分页功能验证 | P1 | 自动化 |
| TD-NOTICE-060 | 权限 | 非admin不可见 | P1 | 自动化 |

## 数据策略
- 测试数据来源：运行时生成唯一标题(时间戳后缀)
- 数据清理方式：fixture teardown中按标题前缀删除
- 前置准备：admin账号登录

## 自动化建议
- 建议自动化：搜索/CRUD — 定位器稳定
- 不建议自动化：富文本编辑器兼容性（手工测试）
- 原因：富文本编辑器iframe结构复杂，Selenium操作不稳定
