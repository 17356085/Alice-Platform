# BUSINESS_SCENARIOS.md

## 业务场景模型 – menu-management（系统管理-菜单管理）

### 1. 业务目标

- **核心业务目标**：维护系统导航菜单的结构和内容，确保不同角色的用户能够看到符合其权限的导航项，并顺畅地进入相应功能页面。
- **次要业务目标**：自定义菜单图标/排序，提升导航的可识别性和可用性。
- **非目标**：不负责用户角色分配（角色权限在 role-management 页面管理），不负责页面内部路由跳转逻辑。

### 2. 角色与旅程

| 角色 | 职责边界 | 典型旅程 |
|------|----------|----------|
| 超级管理员（admin） | 所有菜单管理操作权限 | 登录→进入系统管理→菜单管理→查看菜单树→编辑系统菜单（如“订单管理”）→保存→退出 |
| 系统管理员（sysadmin） | 仅管理非系统级菜单（业务自定义菜单） | 登录→菜单管理→在“应用中心”下新增一个业务菜单→配置名称/路由/图标→勾选“可见角色”→保存→刷新菜单树验证 |
| 普通管理员（normal） | 无菜单管理权限，但可能被分配查看菜单列表的权限 | 登录→点击“系统管理”发现菜单管理不可见或被禁用→退出 |

### 3. 业务流程

| 流程ID | 流程类型 | 描述 | 涉及页面 | 关联风险 |
|--------|----------|------|----------|----------|
| WK-MENU-01 | 主流程（Happy Path） | 超级管理员创建一级菜单“数据报表” → 保存 → 立即在菜单树下出现 → 刷新后依然存在 | login → menu-management（新增弹窗）→ 再次查看菜单树 | RISK-MENU-006, 007 |
| WK-MENU-02 | 主流程（Happy Path） | 超级管理员为“系统管理”添加子菜单“菜单配置”→ 设置路由/权限 → 保存 → 切换到另一个账号登录可看到新菜单 | menu-management → role-management（关联权限）→ login（切换账号）| RISK-MENU-003, 004 |
| WK-MENU-03 | 异常流程（Alternative Path） | 删除一个正在被角色引用的菜单 → 弹窗提示“该菜单已被角色‘运营组’引用，请先解除引用” → 用户取消删除 | menu-management（删除确认框） | RISK-MENU-001 |
| WK-MENU-04 | 异常流程（Alternative Path） | 在已有“用户管理”的父菜单下，再次创建同名“用户管理”菜单 → 后端返回错误“同级菜单名称不能重复” → 前端红字提示 | menu-management（新增弹窗） | RISK-MENU-006 |
| WK-MENU-05 | 分支流程 | 用户拖拽“订单管理”到“系统管理”下面 → 点击“保存排序”按钮 → 菜单树重新加载 → 排序生效 | menu-management（拖拽交互） | RISK-MENU-010 |

### 4. 业务规则

| 规则ID | 规则描述 | 可验证条件 | 对应风险 |
|--------|----------|------------|----------|
| BR-MENU-001 | 菜单层级最多3级（一级→二级→三级） | 选择父菜单时，若选择二级菜单作为父菜单，则新建的菜单为三级；若选择三级菜单作为父菜单，则提示“该菜单已达最大层级” | RISK-MENU-002 |
| BR-MENU-002 | 同一父菜单下菜单名称不能重复（忽略前后空格） | 尝试在“系统管理”下创建两个名为“菜单管理（前后有空格）”的菜单，只有第一个成功 | RISK-MENU-006 |
| BR-MENU-003 | 菜单路由全局唯一，不区分大小写 | 路由“/order”已存在，尝试创建路由“/Order”会失败 | RISK-MENU-007 |
| BR-MENU-004 | 删除菜单前需检查该菜单是否被任何角色引用；若被引用则禁止删除，并列出引用角色名称 | 为“分析报表”菜单关联“运营角色”后，尝试删除该菜单，弹出提示框中列出“运营角色” | RISK-MENU-001 |
| BR-MENU-005 | 菜单名称长度 ≤ 50，路由长度 ≤ 200 | 输入51个字符名称会被前端或后端拒绝，并提示具体长度 | RISK-MENU-005 |

### 5. 数据流

| 数据项 | 来源 | 变换 | 消费端 |
|--------|------|------|--------|
| 菜单树结构 | 数据库 `sys_menu` 表 | 前端递归渲染 `el-tree` | 页面侧边栏导航组件（sidebar） |
| 菜单名称 | 新增/编辑表单输入 | 后端去空格、校验唯一性 | `sys_menu` 表，角色权限分配页面（role-management） |
| 菜单路由 | 新增/编辑表单输入 | 后端校验唯一性、格式化（小写） | 前端 `vue-router` 注册 |
| 菜单与角色关联 | 角色管理页面 | 通过中间表 `role_menu` 存储 | 菜单管理删除时反向检查引用 |
| 排序值 | 拖拽排序后保存 | 后端更新所有同级菜单的 `order_num` | 菜单树渲染时排序 |
