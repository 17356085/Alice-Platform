# AUTO_STRATEGY — system / login-log

## 自动化目标
- 覆盖登录日志页 P0 全量 + P1 ≥80%，预计 10 条自动化用例（只读页面）

## 推荐策略
- 脚本层级：LoginLogPage → conftest.py → test_login_log_management.py
- 断言层级：页面级(元素可见/表格数据) + 交互级(弹窗打开/关闭)
- 数据策略：依赖系统已有登录记录，无需造数据
- 清理策略：不清理（只读页面）

## 适合自动化的场景
- 搜索/状态筛选 — radio/input定位器A级
- 详情弹窗 — 固定标题"登录日志详情"
- 清空弹窗交互 — 仅验证弹窗+取消，不实际清空

## 不适合自动化的场景
- 时间范围搜索边界值验证 — 需大量历史数据
- 导出文件内容验证 — 需人工检查Excel

## 依赖条件
- 环境：test环境，admin账号
- 数据：系统至少有一些登录记录
- 权限：仅admin可访问
