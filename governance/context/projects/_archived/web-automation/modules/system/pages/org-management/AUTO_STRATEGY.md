# AUTO_STRATEGY — system / org-management

## 自动化目标
- 覆盖组织管理页 P0 全量 + P1 ≥80%，预计 12 条自动化用例

## 推荐策略
- 脚本层级：OrgManagePage → conftest.py → test_org_management.py
- 断言层级：页面级(元素可见/表格行数) + 数据级(名称校验) + 流程级(CRUD全链路)
- 数据策略：运行时生成唯一名称(时间戳后缀)，teardown中按前缀清理
- 清理策略：teardown中删除本轮新增测试数据

## 适合自动化的场景
- 搜索功能 — 输入框+按钮定位器A级
- CRUD全流程 — 表单字段固定
- 树结构操作 — 懒加载+展开折叠可编程控制

## 不适合自动化的场景
- 大量部门树性能 — 一次性性能测试
- 导出文件内容验证 — 需人工检查Excel

## 依赖条件
- 环境：test环境，admin账号
- 权限：需admin账号（用于新增/删除/编辑）
