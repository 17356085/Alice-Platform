# AUTO_STRATEGY — system / system-log

## 自动化目标
- 覆盖系统日志页 P0 全量 + P1 ≥80%，预计 9 条自动化用例（只读页面）

## 推荐策略
- 脚本层级：SystemLogPage → conftest.py → test_system_log_management.py
- 断言层级：页面级 + 交互级(下拉筛选/弹窗) + 数据级
- 数据策略：依赖系统已有运行日志，无需造数据
- 清理策略：不清理（只读页面）

## 适合自动化的场景
- 下拉筛选(日志类型/级别) — el-select标准模式
- 搜索 — 输入框定位器A级
- 清空弹窗交互 — 仅验证弹窗+取消

## 不适合自动化的场景
- 日志内容语义分析 — 需人工判断
- 大量数据性能 — 一次性性能测试

## 依赖条件
- 环境：test环境，admin账号
- 数据：系统至少有一些运行日志
- 权限：仅admin可访问
- 接口：日志类型/级别枚举接口需正常返回
