# TEST_DESIGN — dcs / point-config

## 基本信息
- 模块：DCS 数据监控（dcs）
- 页面：点位配置（point-config）
- 测试目标：覆盖点位CRUD、告警规则配置、搜索/筛选/分页全流程
- 需求来源：PAGE_CONTEXT.md + RISK_MODEL.md + PointConfigPage.py

## 测试范围
- 包含：页面加载、搜索/筛选/重置、新增/编辑/删除、告警规则配置、分页导航、类型筛选
- 不包含：导入导出文件格式校验、大数据量性能

## 数据策略
- 测试数据来源：系统已有点位数据 + 测试前缀点位
- 数据清理方式：新增的测试点位按前缀"TEST-"或"TC-"删除
- 前置准备：确保 admin 账号有完整权限

## 自动化建议
- 建议自动化：搜索/分页 — 定位器稳定
- 谨慎自动化：新增/编辑/告警配置 — 弹窗表单复杂，需E2E确认按钮存在
- 不建议自动化：导入格式校验（文件系统依赖）

## 已知限制
- 页面可能为只读查询页，无「新增」「编辑」按钮（test_004/test_005/test_006 已 skip）
- 搜索按钮 DOM 可能与 common-data 不同（test_002/test_003 已 skip）
- 需 E2E 确认 CRUD 按钮是否真实存在
