### 自动化优先级建议

- **P0 (阻塞上线)**: 必须100%自动化。包括：页面加载、新增、删除、权限校验、关键业务场景E2E。
- **P1 (核心体验)**: 建议 ≥80% 自动化。包括：搜索、编辑、边界值校验、表单验证、分页。
- **P2 (边缘场景)**: 建议 ≥50% 自动化。包括：特殊字符搜索、批量删除、取消操作。
- **不建议自动化**:
    1. **无权限的URL访问测试** (如TC-ROLE-LOAD-002): 测试环境配置复杂，手动验证更可靠。
    2. **复杂的UI/UX视觉测试** (如像素级布局检查): 投入产出比低，依赖特定解析器。
    3. **手动构造的、涉及跨系统数据一致性测试** (如删除角色后，关联的审批流如何被影响): 手动操作可以更灵活地观察和验证系统行为。

### 数据清理约定

- **所有创建的测试数据名称统一前缀**：`TC-`
- **Fixture (Teardown)**：所有创建了测试数据的用例，其对应的fixture需要在`teardown`中调用`delete_role_by_name("TC-")`进行清理。
- **清理失败处理**：清理操作如果抛出异常（如API超时），仅记录warning日志，不导致测试用例FAIL。
