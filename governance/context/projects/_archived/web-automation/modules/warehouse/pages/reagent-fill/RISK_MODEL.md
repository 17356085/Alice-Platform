# Risk Model: reagent-fill (三剂消耗-装填管理)

---

## Part 1: RISK_MODEL — 6-Dimension Risk Analysis

### 1.1 Business Risks

| 风险ID | 描述 | 严重度 | 缓解措施 |
|--------|------|--------|----------|
| BR-RF-01 | 新增装填记录未成功保存，业务数据丢失 | P0 | `test_add_item_success` 验证行存在 + count增加 |
| BR-RF-02 | 删除操作误删数据 | P0 | `test_delete_created_item` 验证行不再存在 |
| BR-RF-03 | 搜索功能异常，无法检索到已有数据 | P1 | `test_search_by_item_name` 覆盖搜索路径 |
| BR-RF-04 | 重置功能异常，搜索条件残留 | P2 | `test_reset_search` 验证重置清除条件 |

### 1.2 Permission Risks

| 风险ID | 描述 | 严重度 | 缓解措施 |
|--------|------|--------|----------|
| PR-RF-01 | 无权限用户仍可看到新增/删除按钮 | P0 | 视觉检查，当前无权限测试覆盖 |
| PR-RF-02 | 按钮权限控制失效导致越权操作 | P1 | 需补充权限测试（当前无覆盖） |
| PR-RF-03 | 数据权限隔离失效，看到其他部门数据 | P1 | 需补充跨部门数据测试（当前无覆盖） |

> Note: No explicit permission tests are defined. Permission testing gap identified.

### 1.3 Data Risks

| 风险ID | 描述 | 严重度 | 缓解措施 |
|--------|------|--------|----------|
| DR-RF-01 | 三剂名称输入超长导致数据截断或报错 | P2 | `_fill_dialog_by_placeholder` 无长度限制处理 |
| DR-RF-02 | 特殊字符（SQL注入/XSS）导致异常 | P1 | 当前测试数据仅使用中文+下划线，未覆盖特殊字符 |
| DR-RF-03 | 空名称提交处理不当，后端报500错误 | P0 | `test_add_empty_required` 检查前端错误提示非空 |
| DR-RF-04 | 名称含前后空格导致搜索/删除匹配失败 | P2 | `delete_item_by_name` 使用精确匹配，可能受空格影响 |

### 1.4 Interface Risks

| 风险ID | 描述 | 严重度 | 缓解措施 |
|--------|------|--------|----------|
| IR-RF-01 | 查询接口超时导致表格无限loading | P1 | 显式等待设置超时时间15s |
| IR-RF-02 | 分页参数错误导致数据重复或缺失 | P2 | `test_pagination_visible` 仅验证控件存在 |
| IR-RF-03 | Token过期导致请求401，页面异常 | P1 | 框架层面需统一处理 token 刷新 |
| IR-RF-04 | 并发操作（多人同时新增/删除）导致数据不一致 | P2 | 需补充并发场景测试（当前无覆盖） |

### 1.5 UI/UX Risks

| 风险ID | 描述 | 严重度 | 缓解措施 |
|--------|------|--------|----------|
| UR-RF-01 | el-dialog Teleport渲染导致 `_fill_dialog_by_placeholder` 定位失败 | P1 | 方法搜索全局可见输入框，不依赖DOM层级 |
| UR-RF-02 | 对话框动画未完成时填写，输入丢失 | P1 | `_fill_dialog_by_placeholder` 先确认visible再操作 |
| UR-RF-03 | 多个"新增"按钮共存导致点击到错误按钮 | P2 | `contains(.,"新增")` 可能匹配多个，需保证页面唯一 |
| UR-RF-04 | 表格列固定导致 `is_row_present` 定位到错误区域 | P2 | 当前未测试列固定场景 |

### 1.6 Performance Risks

| 风险ID | 描述 | 严重度 | 缓解措施 |
|--------|------|--------|----------|
| PERF-RF-01 | 大数据量（>10000条）时分页加载缓慢 | P2 | 自动化仅在小数据集验证功能正确性 |
| PERF-RF-02 | 频繁搜索（每秒多次）导致接口雪崩 | P2 | 测试间隔默认等待，不构成性能压力 |
| PERF-RF-03 | 删除操作后分页数据不同步 | P1 | 删除后验证行不在，未验证分页总数更新 |

## Part 2: BUSINESS_SCENARIOS

### 2.1 Business Goal

管理三剂（催化剂、助剂、添加剂）的装填消耗记录，实现装填数据的增删查管理，确保物料消耗可追溯。

### 2.2 Roles & Journeys

| 角色 | 操作旅程 | 涉及功能 |
|------|----------|----------|
| 仓库管理员 | 登录 → 导航至装填管理 → 查询记录 → 新增装填 → 提交保存 | 搜索，新增 |
| 仓库主管 | 登录 → 导航至装填管理 → 浏览记录 → 删除错误数据 | 搜索，删除 |
| 审计人员 | 登录 → 导航至装填管理 → 查看历史记录 | 搜索，浏览 |

### 2.3 Workflows

#### Happy Path: 新增装填记录
```
1. 导航至装填管理页
2. 点击"新增"按钮
3. 对话框弹出，填写三剂名称
4. 点击"确定"提交
5. 表格新增一行，数据保存成功
```

#### Alternative Path: 取消新增
```
1. 导航至装填管理页
2. 点击"新增"按钮
3. 对话框弹出
4. 点击"取消"按钮
5. 对话框关闭，无新数据行
```

#### Exception Path: 必填校验
```
1. 导航至装填管理页
2. 点击"新增"按钮
3. 对话框弹出，不填写内容
4. 点击"确定"提交
5. 显示错误提示，表单未提交
```

#### Happy Path: 搜索删除
```
1. 使用三剂名称搜索
2. 表格过滤出对应行
3. 点击删除操作
4. 确认后数据行消失
```

### 2.4 Business Rules

| 规则ID | 描述 | 验证点 | 置信度 |
|--------|------|--------|--------|
| BR-RF-01 | 三剂名称为必填项 | `assert error != ""` | 高 |
| BR-RF-02 | 新增装填记录必须产生新行 | `after >= before + 1` | 高 |
| BR-RF-03 | 删除装填记录必须移除行 | `not is_row_present` | 高 |
| BR-RF-04 | 取消操作不产生持久化数据 | `not is_row_present` | 高 |
| BR-RF-05 | 按名称搜索应精确匹配 | 搜索后验证目标行可见 | 高 |

### 2.5 Data Flow

```
用户输入 → el-input (v-model) → 前端表单对象 → API POST /api/reagent-fill
                                                      ↓
后端校验 → 数据库写入 ← 审批逻辑 (无) → 返回成功响应
                                                      ↓
前端刷新 → el-table 重新渲染 → 分页更新
```

- 无审批流程（与 spare-in-order 的关键区别）
- 数据流为直接 CRUD
