# 自动化策略：备品盘点调整

- **模块**: `warehouse`
- **页面**: `spare-stock-adjust` (SpareStockAdjustPage)
- **类型**: 只读查询列表（无增删改，无审批流）

---

## 1. 自动化覆盖矩阵

| 用例编号 | 标题 | 优先级 | 是否自动化 | 理由 |
|----------|------|--------|------------|------|
| TC-SA-001 | 页面正常加载（表格渲染） | P0 | ✅ 已自动化 | 基础冒烟，定位器 `TABLE_ROWS` 为 B 级，稳定 |
| TC-SA-002 | 分页器可见 | P0 | ✅ 已自动化 | 组件可识别，定位器 `TOTAL_COUNT` 为 A 级，稳定 |
| TC-SA-003 | 按物品编号搜索 | P1 | ✅ 已自动化 | 核心搜索功能，使用"test"关键字冒烟，B 级定位器 |
| TC-SA-004 | 重置搜索 | P1 | ✅ 已自动化 | 基础功能，当前无显式断言，建议增强 |
| （无） | 按日期查询 | P2 | ❌ 不建议 | 数据动态，断言困难；单日搜索 ROI 低 |
| （无） | 组合搜索（编号+日期） | P2 | ❌ 不建议 | 组合场景手工验证更高效 |

### 风险标注
- 无 C 级定位器
- 所有定位器均为 A/B 级，风险低

### 不自动化理由汇总
- 日期查询：数据高度动态，断言"结果正确"需要强依赖后端预期，成本远高于收益
- 组合搜索：手工执行仅需 10 秒，自动化维护成本较高

---

## 2. PageObject 拆分方案

当前页面极其简单，无需拆分。建议保持一个 Page 类：

| Page 类 | 职责 |
|---------|------|
| `SpareStockAdjustPage` | 搜索区操作（物品编号/日期输入、查询、重置）、表格行读取、分页信息获取 |

**无独立弹窗/对话框**，因此不涉及额外类。

若未来增加"导出"功能（如 `el-button` 触发下载），可保留在当前类中。

---

## 3. 公共组件复用分析

| 操作 | 是否可复用 BasePage 方法 | 方法名 / 说明 |
|------|--------------------------|---------------|
| 导航到页面 | ✅ 由 fixture 封装 | `BasePage.navigate_to()` + 子类 `navigate()` |
| 等待 Vue 稳定 | ✅ | `BasePage.wait_vue_stable()` |
| 等待加载遮罩消失 | ✅ | `BasePage._wait_loading_gone()` |
| 点击按钮 | ✅ | `BasePage.click()` |
| 输入文本 | ✅ | `BasePage.input_text()` |
| 获取表格行 | ✅ | `driver.find_elements(TABLE_ROWS)` |
| 日期选择器操作（当前测试未使用） | ❌ BasePage 无对应方法 | 需要扩展 `ElementPlusHelper` 或自行封装 |

**建议**：若后续需自动化日期查询，可在 `BasePage` 或新工具类中加入 `el-date-picker`（单日选择器）的通用操作。

---

## 4. 等待策略建议

| 场景 | 等待条件 | 建议封装 |
|------|----------|----------|
| 页面加载完成 | 表格至少有一行（或空数据占位）且加载遮罩消失 | `_wait_page_ready()` 已存在 ✅ |
| 输入物品编号后点击查询 | 加载遮罩消失 + Vue 稳定 | `search_by_item_code()` 当前仅调用 `wait_vue_stable()`，建议补充 `_wait_loading_gone()` |
| 点击重置后 | 加载遮罩消失 + Vue 稳定 | `reset_search()` 当前仅调用 `wait_vue_stable()`，建议补充 `_wait_loading_gone()` |
| 日期选择操作（若未来需要） | 下拉面板出现 | 需显式等待 `visibility_of_element_located(body > .el-popper)` |

## 5. ROI 分析

| 项目 | 估计值 | 计算说明 |
|------|--------|----------|
| 自动化开发时间 | 1.5 小时 | 已有 Page Object + 测试脚本，仅需微调 |
| 月维护成本 | 0.3 小时/月 | 定位器极少变动 |
| 手工单次执行时间 | 2 分钟 | 点击页面 + 查看结果 |
| 手工执行频率 | 1 次/天 (20天/月) | 假设每日冒烟执行 |
| **年度 ROI** | **≈ + 35.4 小时** | `(2min × 20 × 12 - 0.3h × 12 - 1.5h) / 60` |

**结论**：自动化覆盖基础冒烟（2个 P0 + 2个 P1）价值明确，开发成本低，维护成本极低，建议立即实施。日期查询场景不纳入自动化（ROI 为负）。

## 附：待改进项

1. **增强重置搜索断言**：当前仅执行重置操作并等待，无任何断言。建议重置后检查物品编号输入框和日期选择器的值是否为空。
2. **补充分页总数断言**：可断言 `TOTAL_COUNT` 的文本不为空且为数字。
3. **补充搜索后等待**：`search_by_item_code` 和 `reset_search` 方法当前仅调用 `wait_vue_stable()`，建议补充 `_wait_loading_gone()` 确保加载遮罩完全消失。
4. **fixture 优化**：确保 `spare_stock_adjust_page` fixture 在浏览器会话中复用，不必每次刷新。
