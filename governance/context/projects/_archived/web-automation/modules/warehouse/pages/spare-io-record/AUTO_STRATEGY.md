# 自动化策略：备品出入库记录

- **模块**: `warehouse`
- **页面**: `spare-io-record` (SpareIORecordPage)
- **类型**: 只读审计日志列表（无增删改，无审批流）

---

## 1. 自动化覆盖矩阵

| 用例编号 | 标题 | 优先级 | 是否自动化 | 理由 |
|----------|------|--------|------------|------|
| TC-SR-001 | 页面正常加载（表格渲染） | P0 | ✅ 已自动化 | 基础冒烟，定位器 `TABLE_ROWS` 为 B 级，稳定 |
| TC-SR-002 | 分页器可见 | P0 | ✅ 已自动化 | 组件可识别，定位器 `TOTAL_COUNT` 为 A 级，稳定 |
| TC-SR-003 | 按物品名称搜索（冒烟） | P1 | ✅ 已自动化 | 核心搜索功能，使用"test"关键字冒烟，B 级定位器 |
| TC-SR-004 | 重置搜索 | P1 | ✅ 已自动化 | 基础功能，当前无显式断言，建议增强 |
| （无） | 空数据状态 | P2 | ❌ 不建议 | 需依赖数据准备，ROI 较低 |
| （无） | 组合搜索（仅一个字段） | P2 | ❌ 不建议 | 页面仅一个搜索字段，无组合场景 |

### 风险标注
- 无 C 级定位器
- 所有定位器均为 A/B 级，风险低

### 不自动化理由汇总
- 空数据状态：需要清理测试数据，维护成本高，手工验证更快
- 当前仅一个搜索字段（物品名称），不存在组合搜索场景

---

## 2. PageObject 拆分方案

当前页面极其简单，无需拆分。建议保持一个 Page 类：

| Page 类 | 职责 |
|---------|------|
| `SpareIORecordPage` | 搜索区操作（物品名称输入/查询/重置）、表格行读取、分页信息获取 |

**无独立弹窗/对话框**，因此不涉及额外类。

若未来增加"导出"功能（如 `el-button` 触发下载），可保留在当前类中。

---

## 3. 公共组件复用分析

| 操作 | 是否可复用 BasePage 方法 | 方法名 / 说明 |
|------|--------------------------|---------------|
| 导航到页面 | ✅ 由 fixture 封装 | `BasePage.navigate_to()` + 子类 `navigate()` |
| 等待 Vue 稳定 | ✅ | `BasePage.wait_vue_stable()` |
| 等待加载遮罩消失 | ✅ | `BasePage._wait_loading_gone()` |
| 点击按钮 | ✅ | `BasePage.click()` |
| 输入文本 | ✅ | `BasePage.input_text()` |
| 获取表格行 | ✅ | `driver.find_elements(TABLE_ROWS)` |

**建议**：本页面无日期选择器，技术复杂度低，所有操作均可直接复用 BasePage 方法。

---

## 4. 等待策略建议

| 场景 | 等待条件 | 建议封装 |
|------|----------|----------|
| 页面加载完成 | 表格至少有一行（或空数据占位）且加载遮罩消失 | `_wait_page_ready()` 已存在 ✅ |
| 输入物品名称后点击查询 | 加载遮罩消失 + Vue 稳定 | `search_by_item_name()` 当前仅调用 `wait_vue_stable()`，建议补充 `_wait_loading_gone()` |
| 点击重置后 | 加载遮罩消失 + Vue 稳定 | `reset_search()` 当前仅调用 `wait_vue_stable()`，建议补充 `_wait_loading_gone()` |

## 5. ROI 分析

| 项目 | 估计值 | 计算说明 |
|------|--------|----------|
| 自动化开发时间 | 1 小时 | 已有 Page Object + 测试脚本，页面结构极简单 |
| 月维护成本 | 0.2 小时/月 | 定位器极少变动 |
| 手工单次执行时间 | 1.5 分钟 | 点击页面 + 查看结果 |
| 手工执行频率 | 1 次/天 (20天/月) | 假设每日冒烟执行 |
| **年度 ROI** | **≈ + 36.6 小时** | `(1.5min × 20 × 12 - 0.2h × 12 - 1h) / 60` |

**结论**：自动化覆盖基础冒烟（2个 P0 + 2个 P1）价值明确，开发成本极低，维护成本极低，建议立即实施。

## 附：待改进项

1. **增强重置搜索断言**：当前仅执行重置操作并等待，无任何断言。建议重置后检查物品名称输入框的值是否为空。
2. **补充分页总数断言**：可断言 `TOTAL_COUNT` 的文本不为空且为数字。
3. **补充搜索后等待**：`search_by_item_name` 和 `reset_search` 方法当前仅调用 `wait_vue_stable()`，建议补充 `_wait_loading_gone()` 确保加载遮罩完全消失。
4. **fixture 优化**：确保 `spare_io_record_page` fixture 在浏览器会话中复用，不必每次刷新。
