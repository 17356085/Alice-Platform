# 业务场景模型: 危险品出库单 (Hazard Outbound Order)

## 1. 业务目标 (Business Goal)

- **核心业务目标 [✅ 已确认]:** **确保危险品出库过程合法、合规、安全、准确。** 系统为库管员、审批人提供一套严格、可追溯的出库流程，防止违规操作、安全事故和库存错误。
- **次要业务目标 [⚠️ 推断]:** **高效处理出库任务。** 通过快捷操作、批量功能、清晰的信息展示，提升库管员和审批人的工作效率。
- **非目标 [✅ 已确认]:** 本页面不处理以下问题：
    - 危险品的运输过程管理（由物流系统处理）。
    - 危险品采购入库（由采购入库单处理）。
    - 非危险品的出库业务。

## 2. 角色与旅程 (Roles & Journeys)

| 角色名 | 职责边界 | 典型旅程 | 多角色协作场景 |
| :--- | :--- | :--- | :--- |
| **库管员 (Warehouse Clerk)** | 创建/编辑/删除并最终执行出库单。 | **旅程1 (Happy Path):** 接收到出库需求 -> 进入页面 -> **新增出库单** -> 填写/选择危险品信息 -> 提交审批 -> 等待审批 -> 审批通过 -> **出库确认** -> 完成实物流转。 | 旅程1 中依赖“审批人”完成审批环节。 |
| **审批人 (Approver)** | 审核出库单的合规性、资质、数量，批准或驳回。 | **旅程2 (审批流程):** 收到待审批通知 -> 进入页面 -> 搜索/查看待审批出库单 -> 点击 **审批** -> 查看出库详情、资质附件 -> 选择“通过”或“驳回” -> 填写审批意见 -> 提交。 | 旅程2 的结果直接影响库管员的后续操作(旅程1)。 |
| **管理员 (Admin)** | 管理基础数据（危险品类别、仓库、用户权限） | **旅程3 (配置管理):** 配置危险品类型 -> 维护仓库信息 -> 设置用户审批权限。 | 库管员和审批人的操作依赖于管理员配置的基础数据。 |

## 3. 业务流程 (Workflows)

### 3.1 端到端主流程：危险品出库

- **流程ID:** WF-ALARM-OUT-001
- **本页面位置:** 子流程节点 `创建并提交`、`审核`、`出库确认`
- **Happy Path:**
    1. **用户（库管员）：** 在系统某处（如生产订单）发起出库需求。
    2. **本页面（创建）：** 跳转到“新增危险品出库单”页面。
    3. **用户（库管员）：** 填写/选择信息（仓库、收发方、危险品明细、关联订单等），上传资质文件。
    4. **系统（预占）：** 预占库存，返回成功信息，刷新出库单列表。
    5. **本页面（列表）：** 新出库单状态为“待审批”。
    6. **用户（审批人）：** 查看“待审批”列表，筛选并点击审批。
    7. **系统（审批）：** 审批通过，状态变更为“已审批”。
    8. **用户（库管员）：** 查看“已审批”列表，找到该出库单。
    9. **用户（库管员）：** 点击“出库确认”，系统/扫码器核实实物。
    10. **系统（扣减）：** 扣减库存，生成出库记录，状态变更为“已出库”。
- **流程结束状态:** `已出库`。

### 3.2 分支流程 (Alternative Paths)

- **Alternative Path 1 (审批被驳回):**
    > 同上流程，但在步骤7，审批人选择“驳回”。
    - **状态流转:** `待审批` -> `已驳回`。
    - **后续操作:** 库管员可查看驳回原因，**编辑**出库单修正后再次提交审批。

- **Alternative Path 2 (取消出库):**
    > 在流程中步骤8(出库确认) 之前，用户发现错误或需求取消。
    - **状态流转 (待审批/已审批):** 通过“删除”按钮取消出库单。
    - **后续操作:** 删除后，预占的库存被释放。

- **Alternative Path 3 (批量操作):**
    > 用户选择多个“待审批”出库单，点击 **批量审批** 或 **批量删除**。

### 3.3 异常恢复流程 (Exception Paths)

| 异常场景 | 触发条件 | 期望行为 | 恢复路径 |
| :--- | :--- | :--- | :--- |
| **创建失败** | 网络中断或后端错误 | 显示错误提示，表单数据不丢失。 | 用户可修正后重新提交。 |
| **预占库存失败** | 库存不足 | 创建失败，提示“库存不足”。 | 用户返回修改数量或取消出库。 |
| **审批时发现数据错误** | 审批人想修改出库单 | 审批操作不支持修改单据。 | 驳回单据，库管员修改后重新提交。 |

## 4. 业务规则 (Business Rules)

| 规则ID | 规则描述 | 校验阶段 | 业务影响 | 自信度 |
| :--- | :--- | :--- | :--- | :--- |
| **RULE-001 (危化品限量)** | 单次出库单的总重量/体积不得超过特定危险品的运输限额(例如，单个易燃液体类别单次出库不超过1000L)。 | 出库单创建 / 出库确认 | 防止超限运输，规避法规风险。 | ⚠️ 推断 |

| **RULE-003 (状态流转)** | 出库单状态只能按 **“待审批” → “已审批” → “已出库”** 或 **“待审批” → “已驳回”** 的顺序流转，不能跳跃或逆向。 | 后端接口 | 保证业务流程的严谨性。 | ✅ 确认 |
| **RULE-004 (库存核验)** | 出库确认时，系统必须再次校验明细中的危险品数量不超过当前仓库的**可用库存量**（考虑了其他订单的预占）。 | 出库确认（后端接口） | 防止超卖、库存负数。 | ✅ 确认 |
