# 风险模型: 备品出库 (Spare Out Order)

## 1. 业务风险 (Business Risk)

| 风险ID | 风险描述 | 等级 | 触发条件 | 影响 | 缓解/验证措施 | 备注 |
|--------|----------|------|----------|------|---------------|------|
| BUSK-SOO-001 | **备品出库数量错误导致库存偏差** | P0 | 出库数量填写错误或重复提交 | 库存台账数据不准确，影响采购计划和生产排期 | 1. 出库单创建时预占库存<br>2. 出库确认时再次校验可用量 | 核心业务风险 |
| BUSK-SOO-002 | **LY 单号链接失效** | P1 | LY 单号链接目标页面 URL 变更或页面重构 | 用户无法通过出库单追溯 LY 单据 | 1. 链接导航使用路由而非硬编码 URL<br>2. 自动化冒烟测试定期验证 | 特殊交互风险 |
| BUSK-SOO-003 | **未遵循审批链即出库** | P0 | 跳过审批流程直接操作出库 | 违反公司内部管控流程，可能导致资产流失 | 1. 审批链强制校验<br>2. 后端状态机限制出库操作 | 关联审批链 |
| BUSK-SOO-004 | **出库单数据可追溯性缺失** | P1 | 删除已出库记录或不保留操作日志 | 审计无法追溯历史出库行为 | 1. 已出库记录禁止删除<br>2. 操作日志永久保存 | 中置信度推断 |
| BUSK-SOO-005 | **备件查询导航后丢失上下文** | P2 | 从备件查询页面返回时出库单列表刷新 | 用户需要重新搜索，体验不佳 | 搜索条件应保持/支持浏览器返回 | 低置信度推断 |

# 业务场景模型: 备品出库 (Spare Out Order)

## 1. 业务目标

- **核心业务目标**: 确保备品出库过程准确、可追溯，出库记录与库存数据实时同步。
- **次要目标**: 提供快捷的 LY 单号追溯和备件查询能力，提升库管员工作效率。

## 2. 角色与旅程

| 角色名 | 职责边界 | 典型旅程 |
|--------|----------|----------|
| **仓库管理员** | 创建/管理出库单，执行出库操作 | 新建出库单 -> 填写经办人 -> 保存 -> 搜索确认 -> 完成出库 |
| **审批人 (admin/chenqian)** | 会签审批出库单 | 查看待审批出库单 -> 审批通过/驳回 |
| **备件查询员** | 查询备件信息 | 点击备件查询按钮 -> 在备件查询页面查看信息 |
| **系统管理员** | 配置审批链和权限 | 配置备件出库审批链 -> 分配权限 |

## 3. 业务流程

### Happy Path
1. 仓库管理员导航到备品出库页面
2. 点击新增，填写经办人信息
3. 保存出库单，状态变为"待审批"
4. admin + chenqian 进行会签审批
5. 审批通过后执行实际出库

### Alternative Path
1. **取消出库**: 填写出库单后取消，记录不保存
2. **删除出库**: 删除草稿/待审批状态的出库单
3. **LY 单号追溯**: 点击 LY 链接查看关联单据详情
4. **备件查询**: 导航到备件查询页面查看备件信息

### Exception Path
1. **必填校验失败**: 不填写经办人直接保存，系统提示必填
2. **删除已审批出库单**: 已审批记录不可删除
3. **接口错误**: 保存失败，弹出错误提示

## 4. 数据流

| 数据项 | 数据来源 | 数据变换 | 数据消费 |
|--------|----------|----------|----------|
| 出库单号（LY 编号） | 创建时系统生成 | 用户在 LY 链接点击跳转关联单据 | LY 单号详情页面 |
| 经办人 | 用户填写 | 作为搜索条件/出库记录信息 | 列表展示/搜索筛选 |
| 库存数量 | 库存台账 | 出库后扣减 | 库存台账页面 |
