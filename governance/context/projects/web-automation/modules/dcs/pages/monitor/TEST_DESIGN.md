# TEST_DESIGN — dcs / monitor

## 基本信息
- 模块：DCS 数据监控（dcs）
- 页面：关键参数监控（monitor）
- 测试目标：覆盖监控仪表盘的卡片加载、状态展示、图表渲染
- 需求来源：PAGE_CONTEXT.md + RISK_MODEL.md + MonitorPage.py

## 测试范围
- 包含：页面加载、监控卡片显示、状态标签验证、图表渲染
- 不包含：新增/编辑/删除（只读仪表盘）、搜索/重置（仪表盘无传统搜索栏）

## 数据策略
- 测试数据来源：系统实时参数数据（DCS推送）
- 数据清理方式：只读页面，无需清理
- 前置准备：确保有活跃的参数数据推送

## 自动化建议
- 建议自动化：页面加载/卡片读取/状态标签 — 验证显示层
- 谨慎自动化：图表验证（canvas内容无法直接断言）/数据刷新（依赖推送频率）
- 不建议自动化：实时数据准确性（后端集成测试范畴）

## 已知限制
- 仪表盘为只读，无传统搜索栏（test_002/test_003 已 skip）
- 无刷新按钮（数据自动推送）（test_004 已 skip）
- 无新增/编辑功能（test_005/test_101 已 skip）
- 卡片选择器需根据实际 DOM 调整（test_006 已 skip）
