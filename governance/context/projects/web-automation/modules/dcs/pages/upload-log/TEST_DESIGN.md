# TEST_DESIGN — dcs / upload-log

## 基本信息
- 模块：DCS 数据监控（dcs）
- 页面：上传日志（upload-log）
- 测试目标：覆盖日志列表的统计卡片、多维筛选、详情查看、分页导航
- 需求来源：PAGE_CONTEXT.md + RISK_MODEL.md + UploadLogPage.py

## 测试范围
- 包含：页面加载、统计卡片验证、日期范围筛选、设备/状态筛选、搜索、详情弹窗、分页
- 不包含：文件导出下载、大数据量性能、清空日志（破坏性操作）

## 数据策略
- 测试数据来源：系统已有的上传日志数据
- 数据清理方式：只读页面，无需清理（避免使用清空按钮）
- 前置准备：确保有足够的历史日志数据（覆盖多种状态）

## 自动化建议
- 建议自动化：统计卡片/状态筛选/分页 — 定位器稳定
- 谨慎自动化：日期范围筛选（日期选择器弹出面板可能遮挡）/详情弹窗（按钮定位器需E2E确认）
- 不建议自动化：清空日志（破坏性操作）/导出下载（文件系统依赖）
