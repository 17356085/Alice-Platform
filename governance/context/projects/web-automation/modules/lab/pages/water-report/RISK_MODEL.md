## 1. 业务风险

### RISK-BIZ-001 (P0) — 误删/误修改已审核报告单
- **描述**: 用户可能在表格中操作“删除”或“编辑”已审核的报告单，导致数据状态错误，或影响下游数据一致性（如已用于排放统计）。
- **缓解措施**: ① 编辑/删除按钮对“已审核”状态报告单置灰或隐藏；② 点击删除前弹出二次确认弹窗，并提示“该报告单已审核，删除后相关统计可能异常”；③ 提供软删除（标记删除）而非物理删除，保留审计日志。
- **自动化覆盖**: 测试脚本未覆盖此场景。`test_water_report.py` 仅验证新增弹窗打开，未验证表格行操作权限。

### RISK-BIZ-002 (P1) — 新增报告单时取样位置与当日取样计划不匹配
- **描述**: 用户可能新增一个不存在的取样位置的报告单，或重复添加同一位置同一时间的报告单，造成数据冗余。
- **缓解措施**: 弹窗中取样位置下拉列表应来源于当日有效取样计划，并唯一约束（同一位置+日期+时间不允许重复）。
- **自动化覆盖**: 无。

### RISK-BIZ-003 (P1) — 导出报告单时数据未包含所有筛选条件
- **描述**: 导出报表可能忽略当前取样位置标签或日期范围，导出的数据与页面上看到的不一致。
- **缓解措施**: 导出接口必须携带当前所有搜索条件参数；前端应在导出前同步查询条件。
- **自动化覆盖**: 导出按钮在代码中未实现，也未出现在测试脚本中。

### RISK-BIZ-004 (P2) — 日期筛选的跨年/跨月逻辑歧义
- **描述**: 用户选择开始日期 >= 结束日期时，系统可能返回空或报错，但未给出明确提示。
- **缓解措施**: 前端校验开始日期 <= 结束日期，否则禁用查询按钮或弹出提示。

## 3. 数据风险

### RISK-DATA-001 (P0) — 水质指标值输入负数或超物理范围（如 pH > 14）
- **描述**: 新增弹窗中“水质指标”字段可能未做范围校验，允许输入极端值，导致报表异常或下游计算溢出。
- **缓解措施**: ① 前端每个指标限制输入最大值/最小值（如 pH 0-14）；② 后端同样校验；③ 如果指标类型为数值且允许小数，限制小数位数。
- **自动化覆盖**: 测试脚本未覆盖指标值输入验证。

### RISK-DATA-002 (P1) — 必填字段未填写时无明确错误提示
- **描述**: 检验员、复核员、取样位置等标有 `*` 的字段未填写时，点击保存后弹窗一闪而过或无反馈。
- **缓解措施**: Element Plus `el-form` 自带校验，确保 rules 配置正确；后端返回 400 时前端应解析错误并显示在对应字段下方。
- **自动化覆盖**: `test_add_dialog_opens` 仅打开弹窗，未测试空表单提交。

### RISK-DATA-003 (P1) — 日期/时间字段输入非法格式或未来日期
- **描述**: 用户可能手动输入格式错误的日期（如 `2026-13-01`）或检验日期为未来日期（取样时间在未来）。
- **缓解措施**: 使用 Element Plus 日期选择器禁止手动输入；或校验日期不能晚于当前时间（取样时间一般 <= 当前时间 + 5分钟容忍）。
- **自动化覆盖**: 无。

### RISK-DATA-004 (P1) — 并发新增：同一检验员在同一时间内创建多条报告单
- **描述**: 两个用户同时操作，可能违反业务约束（同一检验员同一时间只能有一份报告单）。
- **缓解措施**: 后端使用数据库唯一索引（检验员 + 取样位置 + 日期）避免重复；前端可禁用按钮防重复点击。
- **自动化覆盖**: 无并发测试。

### RISK-DATA-005 (P2) — 表格中报告单号超长截断未显示完整
- **描述**: 报告单号如果为自动生成长字符串（如包含时间戳 + 随机码），在表格中可能被省略号截断，无法完整查看。
- **缓解措施**: 表格列设置 `show-overflow-tooltip` 或固定宽度留足空间；操作列提供“查看详情”或复制功能。
- **自动化覆盖**: 无。

### RISK-DATA-006 (P2) — 清空日期选择器后查询显示所有数据
- **描述**: 用户手动清除日期输入框后点击查询，可能查询到所有历史数据（包括未限制的），导致数据量过大。
- **缓解措施**: 日期为空时默认查询最近一个月，或在查询时强制传递默认范围。
- **自动化覆盖**: 无。

## 4. 接口风险

### RISK-API-001 (P0) — 新增报告单接口返回成功但实际数据未插入
- **描述**: 由于后端事务回滚但返回 200，前端 showing 保存成功提示，实际数据未入库，用户后续无法搜索到该报告单。
- **缓解措施**: ① 后端接口返回 `{ code: 0, data: { id: ... } }` 而非简单 `success`；② 前端在保存成功后立即调用一次详情接口或刷新表格验证；③ 添加失败重试及告警。
- **自动化覆盖**: 无。

### RISK-API-002 (P1) — 查询接口分页参数错误导致前端展示不全
- **描述**: 自定义 `report-table` 分页参数（page, size）未正确传给接口，或后端返回总条数与实际不符，导致最后一页无数据或重复。
- **缓解措施**: ① 前端每次查询都携带 `page=1, size=20` 等默认参数；② 接口返回 `total` 字段，前端根据 total 渲染分页；③ 测试时覆盖切换页面、改变每页条数等场景。
- **自动化覆盖**: 测试脚本中 `search_by_date` 未检查分页。

### RISK-API-003 (P1) — 接口超时/500 错误时前端卡死
- **描述**: 网络延迟或服务错误时，页面 loading 一直旋转，用户无法进行其他操作。
- **缓解措施**: ① 所有请求设置超时（如 10s）；② 异常时显示友好错误提示并关闭 loading；③ 对于列表接口，保留旧数据而非清空。
- **自动化覆盖**: 无。

## 5. UI/UX 风险

### RISK-UI-001 (P1) — 切换取样位置后表格数据未刷新
- **描述**: 用户点击取样位置标签，页面 URL 不变，可能未触发数据重新加载，仍显示之前位置的数据。
- **缓解措施**: 标签切换事件绑定 `change` 回调，自动调用 `click_query()` 根据新位置刷新表格。
- **自动化覆盖**: `test_switch_location` 仅验证切换，未验证表格内容是否更新。

### RISK-UI-002 (P1) — 新增弹窗关闭后残留表单数据
- **描述**: 用户第一次打开弹窗填写部分字段后取消，再次打开弹窗时上次的数据仍留在表单中。
- **缓解措施**: 弹窗关闭时重置表单（`el-form.resetFields()`）；或每次打开时调用 `resetFields`。
- **自动化覆盖**: 未测试二次打开弹窗时表单是否清空。

### RISK-UI-003 (P2) — 浏览器缩放或低分辨率下加号按钮/日期选择器遮挡
- **描述**: 在 1366×768 分辨率下，搜索区的“新增”按钮可能被日期选择器弹层覆盖。
- **缓解措施**: 使用 CSS `z-index` 和响应式布局；日期选择器 `append-to-body` 设置为 true。
- **自动化覆盖**: 无。

### RISK-UI-004 (P2) — 表格行高亮无法区分选中状态
- **描述**: 点击行后未添加高亮背景，用户不清楚当前选择了哪一行（若存在批量操作）。
- **缓解措施**: 表格配置 `highlight-current-row`；若无需行选中则无需处理。
- **自动化覆盖**: 无。

### RISK-UI-005 (P2) — 提示文案不统一（中英文混排）
- **描述**: 部分错误提示为后端返回的英文错误（如 `Internal Server Error`），未翻译。
- **缓解措施**: 前端统一拦截 HTTP 错误并显示中文提示；后端返回错误码而非直接抛出异常。
- **自动化覆盖**: 无。

## 6. 性能风险

### RISK-PERF-001 (P1) — 大数据量表格渲染卡顿（>5000 行）
- **描述**: 水质分析报告单可能累积大量历史数据，一次性加载全部数据导致页面卡顿。
- **缓解措施**: ① 默认分页（每页 20 条）；② 后端必须支持分页，前端使用 `:pager`；③ 若用户需要全量导出，走异步导出任务。
- **自动化覆盖**: 无。

### RISK-PERF-002 (P2) — 频繁点击查询按钮触发过多请求
- **描述**: 用户快速多次点击查询，可能发送多个请求，后端压力大且前端数据错乱（后返回的结果覆盖先返回的）。
- **缓解措施**: 点击查询后立即禁用按钮 1s 或使用 loading 状态阻止双击；请求使用 `cancelToken` 取消旧请求。
- **自动化覆盖**: 无。
