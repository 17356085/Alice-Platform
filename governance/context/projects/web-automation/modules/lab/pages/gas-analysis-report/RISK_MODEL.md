### BUSINESS_SCENARIOS.md: 气体分析报告业务场景模型

> 本文件描述了业务上要验证的核心场景，不描述具体的UI操作。

#### 1. 业务目标 (Business Goal)

-   **核心业务目标**：为炼油/化工/钢铁等行业的实验室提供一份标准化、可追溯、权威的气体成分分析结果，用于指导工艺参数调整（如配气比例）或产品质量合格判定。
-   **次要业务目标**：提供历史报告（如每日/每批次）的查询、比对功能，用于生产稳定性分析和问题溯源。
-   **非目标**：该页面不直接控制分析仪器（仪器控制由另一个独立模块负责），也不生成财务对账数据。

#### 2. 角色与旅程 (Roles & Journeys)

| 角色 | 职责边界 | 典型旅程 |
|------|----------|----------|
| **分析员 (Analyst)** | 负责执行分析任务，录入/导入数据，生成和提交报告草稿。 | 1. 进入“气体分析报告”页 -> 2. 点击“新建报告” -> 3. 选择关联样本批次、分析仪器 -> 4. 手动录入或导入仪器数据 -> 5. 点击“保存” -> 6. 点击“提交审批”。 |
| **审核员 (Reviewer)** | 负责审核分析结果的准确性，可退回修改或通过审核。 | 1. 进入报告列表，筛选“待审核”状态的报告 -> 2. 点击查看报告详情 -> 3. 复核数据，与仪器日志比对 -> 4. 点击“通过”或“退回”。 |
| **管理员 (Admin)** | 负责配置报告模板、管理数据字典、查看系统日志和所有报告。 | 1. 进入“气体分析报告”页 -> 2. 点击“报告配置” -> 3. 修改数据展示格式或单位 -> 4. 查看审计日志，追踪谁修改了哪个报告。 |

#### 3. 业务流程 (Workflows)

-   **Happy Path: 标准分析报告生成与审核流程**
    `新建报告 -> 选择仪器 -> 录入/导入数据 -> 保存草稿 -> 提交审批 -> 审核员审批通过 -> 报告状态变更为“已审核” -> 自动归档。`
    -   本页面在流程中的位置：从“新建报告”到“提交审批”的部分。
-   **Alternative Path 1: 报告被退回**
    `提交审批 -> 审核员审批不通过 -> 报告状态变为“已退回” -> 分析员收到通知 -> 修改报告内容 -> 再次提交审批。`
-   **Alternative Path 2: 历史报告对比**
    `报告列表 -> 勾选两份“已审核”报告 -> 点击“对比分析” -> 系统展示数据差异列表 -> 用户导出对比结果。`

#### 4. 业务规则 (Business Rules)

| 规则ID | 规则描述 | 置信度 |
|--------|----------|--------|
| RULE-GAR-001 | **报告状态机规则**：报告状态只能按照 `草稿 → 待审批 → 已审核 → 已归档` 或 `待审批 → 已退回 → 草稿` 的顺序流转，不可跳级。 | ✅ |
| RULE-GAR-002 | **审批权限规则**：一个报告不能被创建它的分析员本人审核。审核员角色必须与报告所属实验室的仪器关联。 | ⚠️ 推断 |
| RULE-GAR-003 | **数据来源约束**：当选择一个仪器后，分析结果数据必须来自该仪器的“已完成分析”记录，不能手动输入与仪器ID不对应的数据。 | ⚠️ 推断 |
| RULE-GAR-004 | **版本控制规则**：任何对“已审核”报告的修改，都必须在详情页触发“创建新版本”操作，原始版本不得被覆盖。 | ✅|

#### 5. 数据流 (Data Flow)

-   **数据来源**:
    -   `仪器管理`模块产生“分析仪器”选择和序列号。
    -   `采样管理`模块产生“样本批次”选择和成分预期。
    -   `实际分析数据`可能来自 `仪器对接API` (自动导入) 或 手动录入。
-   **数据变换**:
    -   本页面组合以上数据源，并根据预设的`报告模板`进行格式化和计算（如计算平均值、标准偏差）。
    -   生成一个最终的`报告对象`（包含基本信息、成分列表、结果判定）。
-   **数据消费**:
    -   `生产管理`模块 (产线MES) 消费“已审核”报告中的最终成分数据，用于调整工艺参数。
    -   `质量管理`模块 (QMS) 消费“已归档”报告，用于月度质量分析报表。
-   **跨页面数据一致性要求**：
    -   页面删除一份报告时，应同步更新 `质量管理` 模块中引用的数据状态。
