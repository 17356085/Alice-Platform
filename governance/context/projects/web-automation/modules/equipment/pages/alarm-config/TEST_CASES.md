## 自动化优先级建议

- **适合自动化**：
  - 所有 P0 场景（TC-LOAD-001, TC-CREATE-001, TC-DEL-001, TC-PERM-001, TC-PERM-003, BS-CREATE-E2E-001, BS-DELETE-RECOVER-001, BS-PERMISSION-001）
  - 大部分 P1 场景（边界值、搜索、分页、权限）
  - 稳定且执行频率高的回归场景
- **不建议自动化**：
  - 依赖外部服务（如短信通知）的端到端验证（可部分模拟）
  - 需要人工视觉判断的UI布局微调（如空状态图片显示）
  - 需要协同多用户同时操作的并发测试
- **混合模式**：
  - 接口自动化 + 少量UI端到端覆盖核心业务流程
  - 异常场景使用mock接口进行自动化

## 自动化优先级建议（合并）

- **P0 用例**：100% 自动化覆盖（已标记 ✅ 已自动化 或 🔄 待开发）
- **P1 用例**：≥80% 自动化覆盖（当前标记中，仅个别因需要外部服务或并发测试暂未自动化）
- **P2 用例**：≥50% 自动化覆盖（简单场景优先自动化，UI布局类暂不自动化）
- **不建议自动化**：
  - TC-LOAD-003（空状态图片展示，人工视觉确认更可靠）
  - 需要真实短信/邮件发送验证的端到端场景（可部分用mock代替）
  - 多用户并发编辑冲突（手动测试更直观）

所有创建的测试数据统一使用前缀 `TC-`，并在 teardown 中通过 `CleanupTracker` 清理，确保不留脏数据。
