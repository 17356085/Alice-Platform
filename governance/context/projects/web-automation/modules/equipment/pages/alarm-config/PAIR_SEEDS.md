# PAIR_SEEDS — 设备报警配置

> 结对同学：AI Pilot Test | 日期：2026-06-22 | 需求ID：TAPD-EQ-ALARM-001

## 种子场景

### SEED-001：报警规则被其他设备引用时不可删除

- **优先级**: P0
- **类型**: negative
- **业务背景**: 实际业务中，一条报警规则可能被多台设备引用。如果允许删除正在被引用的规则，会导致引用设备的监控失效，属于生产事故。AI 从页面分析只能看到"删除按钮+确认弹窗"，不知道后端有引用校验逻辑。
- **前置条件**: 1. 已存在一条报警规则 "高温报警"，阈值 80°C 2. 该规则已被设备 "1号储罐" 和 "2号储罐" 引用
- **操作步骤（概要）**:
  1. 打开设备报警配置页面
  2. 在表格中找到 "高温报警" 规则
  3. 点击该行的"删除"按钮
  4. 在确认弹窗中点击"确定"
- **预期结果**: 前端 toast 提示"该规则正被 2 台设备引用，无法删除"。表格中该规则仍然存在。
- **关键检查点**: 弹窗确认后不是直接删掉，而是后端返回引用检查结果。需验证 toast 消息内容和规则仍然在表格中。

### SEED-002：启用/禁用状态切换后搜索列表中正确反映

- **优先级**: P0
- **类型**: positive
- **业务背景**: 运维人员在排查报警误报时，需要先禁用规则再观察。禁用后必须能在列表中看到状态变化，并且"状态"筛选器能正确过滤。这是运维日常操作，但 AI 常忽略"搜索筛选联动"这个维度。
- **前置条件**: 1. 表格中有至少 2 条规则，其中 1 条状态为"启用" 2. 状态筛选器默认选中"全部"
- **操作步骤（概要）**:
  1. 在表格中找到一条"启用"状态的规则
  2. 点击该行的"禁用"按钮
  3. 观察该行状态列是否变为"禁用"
  4. 在搜索区将"状态"筛选器切换为"禁用"
  5. 点击"搜索"
  6. 切换回"启用"筛选，再次搜索
- **预期结果**: 1. 禁用操作后行状态即时更新 2. "禁用"筛选结果中显示该规则 3. "启用"筛选结果中不包含该规则
- **关键检查点**: 状态切换是即时生效还是需要刷新页面？筛选器联动是否正确？

### SEED-003：报警阈值边界 — 负数输入

- **优先级**: P1
- **类型**: boundary
- **业务背景**: 某些传感器（如液位）的报警阈值不允许为负数。但前端如果没有 `min="0"` 校验，用户可以输入负数并提交，导致后端报错或数据异常。AI 通常会测试"空值"和"超长文本"，但负数这个边界容易被遗漏。
- **前置条件**: 1. 已登录有新增权限的账号 2. 报警类型为"液位报警"（阈值类型=数字）
- **操作步骤（概要）**:
  1. 点击"新增"按钮打开弹窗
  2. 选择报警类型为"液位报警"
  3. 在触发值输入框中输入 "-50"
  4. 填写其他必填项
  5. 点击"确定"提交
- **预期结果**: 前端校验拦截，toast 提示"触发值不能为负数"或输入框标红，表单不提交。
- **关键检查点**: 前端是否有 `min` 属性校验？不同报警类型的阈值范围是否不同？

## 补充说明

- **特殊业务规则**: 报警类型决定阈值输入框的数据类型——温度/压力/液位为数字，状态报警为下拉选项。新增时需根据类型动态切换输入控件。
- **已知坑点**: el-cascader 级联选择器（如果设备选择用了它）需等待异步加载子节点，否则 `NoSuchElementException`。el-date-picker 的范围选择器，DOM 渲染在 body 层。
- **不需要测试的场景**: 纯 UI 美观度验证（颜色/字号/间距），不同分辨率下的布局（P2，CI 环境固定分辨率 1920x1080）。
