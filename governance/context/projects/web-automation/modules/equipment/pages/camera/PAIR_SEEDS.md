# PAIR_SEEDS — 摄像头管理

> 结对同学：AI Pilot Test | 日期：2026-06-22 | 需求ID：TAPD-EQ-CAMERA-001

## 种子场景

### SEED-001：实时预览加载失败时的降级展示

- **优先级**: P0
- **类型**: negative
- **业务背景**: 生产环境摄像头可能因网络/硬件故障无法拉流。AI 易忽略"预览失败"这个异常路径，只测正常加载。运维人员需要看到明确的失败提示而非白屏或无限加载。
- **前置条件**: 1. 已配置至少 1 个摄像头 2. 模拟该摄像头 RTSP 流不可达
- **操作步骤（概要）**:
  1. 打开摄像头管理页面
  2. 点击某个离线摄像头的"预览"按钮
  3. 等待 10 秒
- **预期结果**: 预览区域显示"设备离线"或"无法连接"提示，而非白屏。不阻塞页面其他操作。
- **关键检查点**: 超时时间是否合理（≤10s）？失败后能否关闭预览回到列表？

### SEED-002：批量选择摄像头进行配置下发

- **优先级**: P0
- **类型**: positive
- **业务背景**: 运维人员经常需要对多台摄像头统一下发配置（如调整录像计划）。AI 通常测单个操作，批量操作容易被遗漏——但这是运维日常最高频的操作之一。
- **前置条件**: 1. 列表中有 ≥3 个摄像头 2. 用户有配置管理权限
- **操作步骤（概要）**:
  1. 勾选 2 个摄像头的 checkbox
  2. 点击"批量配置"按钮
  3. 在弹出的配置表单中修改"录像计划"为"全天录像"
  4. 点击"确定"
- **预期结果**: 两个摄像头的配置同时更新。列表刷新后两个摄像头的状态显示为"全天录像"。
- **关键检查点**: 配置下发是同步还是异步？失败时是否有部分成功的提示？

### SEED-003：设备状态筛选联动

- **优先级**: P1
- **类型**: boundary
- **业务背景**: 摄像头状态有"在线/离线/故障"三种，AI 常测试"全部"和单个状态，但忽略"连续切换筛选条件"——快速切换时可能出现 DOM 未更新、数据错乱。
- **前置条件**: 列表中有在线和离线摄像头各若干
- **操作步骤（概要）**:
  1. 筛选器选择"在线" → 搜索
  2. 立即切换为"离线" → 搜索
  3. 再切回"全部" → 搜索
- **预期结果**: 每次切换后表格数据与筛选条件一致。无数据错乱、无闪烁、无重复行。
- **关键检查点**: 快速连续切换时是否有 loading 状态冲突？

## 补充说明

- **特殊业务规则**: 摄像头配置修改后需要设备重启才生效（部分型号）。页面应有"配置已保存，重启后生效"的提示。
- **已知坑点**: 视频流使用 `<video>` 标签，Selenium 无法直接检测视频是否在播放——需用 JavaScript 检查 `video.readyState`。
- **不需要测试的场景**: 视频画质/清晰度验证（人工视觉判断），不同分辨率视频流的兼容性（硬件相关）。
