# BUSINESS_SCENARIOS — 摄像头管理 (推测版)

> **场景映射目的**: 从真实业务流出发，确定测试的覆盖范围和优先级。本文件是 `testcase-design` 的必选输入。

## 1. 业务目标 (Business Goal)

- **核心业务目标**: 授权用户能够高效地**管理**（增、删、改、查、分组）组织内的所有IP摄像头，确保监控系统稳定可靠。
- **次要业务目标**: 提供直观的**状态监控**（在线/离线/异常），帮助运维人员快速定位并解决问题。
- **非目标**: 本页面不负责实时视频流的解码或显示（由专门的播放器页面或组件负责）。不负责处理摄像头的固件升级。

## 2. 角色与旅程 (Roles & Journeys)

## 3. 业务流程 (Workflows)

### 主流程: 添加并配置新摄像头

- **Happy Path (BS-WF-001)**:
  1. 用户A（管理员）登录系统。
  2. 导航到「设备管理 → 摄像头管理」。
  3. 点击「添加摄像头」按钮。
  4. 在Dialog中填写：名称、IP地址、端口号、所属分组（下拉选择）。
  5. 点击「保存」。
  6. 页面刷新，列表顶部出现新添加的摄像头，状态为「离线」（或「等待激活」）。
  7. 系统自动尝试与摄像头建立连接。
  8. 连接成功后，状态更新为「在线」。
- **Alternative Path 1 (BS-WF-002)**: 添加失败——必填项未填。
- **Alternative Path 2 (BS-WF-003)**: 添加失败——IP地址冲突。

### 分支流程: 分组管理

- **BS-WF-004**: 用户进入「编辑摄像头」弹窗，修改其「所属分组」。
- **BS-WF-005**: 用户进入「摄像头管理」首页，使用顶部的「分组」下拉选择器（`group-select`）来筛选展示特定分组的摄像头。
