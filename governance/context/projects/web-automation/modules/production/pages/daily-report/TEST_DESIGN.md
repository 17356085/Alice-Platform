# TEST_DESIGN — production / daily-report

## 基本信息
- 模块：生产管理（production）
- 页面：日报表管理（daily-report）
- 测试目标：覆盖日报表的日期查询、数据展示（4分区表格）、录入/补录/趋势/导出弹窗交互
- 需求来源：PAGE_CONTEXT.md + RISK_MODEL.md

## 测试范围
- **包含**：页面加载、日期查询、4个分区表格数据展示、录入/补录/趋势/导出弹窗交互、按钮状态
- **不包含**：打印功能（浏览器原生对话框）、调整数据（功能可能未实现，默认 disabled）、数据准确性验证（属于后端测试）、权限验证（暂无权限账号）

## 数据策略
- **测试数据来源**：使用系统已有生产数据（当日或近7天内数据）
- **数据清理方式**：录入数据操作为只读型测试，不创建新数据（确保录入弹窗确认前关闭）
- **前置准备**：admin 账号有完整权限；确保所选日期存在生产数据
- **⚠️ 注意**：TD-PROD-DR-008（录入数据成功）会实际写入数据，默认跳过「确定」点击，仅验证弹窗交互。如需要执行完整录入测试，需配套数据清理策略

## 自动化建议
- **Fixture 策略**：module 级 driver_setup（已有 `script/production/conftest.py`）
- **等待策略**：导航后等待 `el-card__header` 出现；查询后等待 loading 遮罩消失
- **定位策略**：按钮使用 XPath 文本匹配；弹窗使用标题 anchor；日期选择使用 placeholder
- **测试顺序**：无依赖关系，各用例独立
- **跳过标记**：录入确认操作用 `@pytest.mark.skip`（避免脏数据）
