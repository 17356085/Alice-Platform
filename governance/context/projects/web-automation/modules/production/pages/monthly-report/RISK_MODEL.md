# BUSINESS_SCENARIOS — production / monthly-report

## 1. 业务目标 (Business Goal)

本页面服务于**生产数据分析**这个核心业务目标，为生产管理者和操作员提供月度生产概况。

- **核心业务目标**: 按月度维度，集中展示LNG、液氮、脱碳气等核心产品的产量、原料消耗、公辅工程指标和冷剂消耗的完整统计报表，为生产绩效评估和成本核算提供数据依据。
- **次要业务目标**: 支持生成、查看历史月报数据趋势，并可将报表导出为文件进行存档和分发，或直接打印。
- **非目标**: 本页面是纯只读的，它不负责**录入、修改或删除**任何生产数据。数据的录入操作在“生产日报表”页面完成。

## 2. 角色与旅程 (Roles & Journeys)

| 角色 | 职责边界 | 典型业务旅程 |
|------|----------|--------------|
| **操作工** | 负责查看本班次/月度末的最终生产成果。 | 1. 登录系统 <br> 2. 进入生产管理 → 生产月报表 <br> 3. 查看当前月份的LNG、液氮等核心产出数值 <br> 4. 查看“产品”分区，核对本月的产量累计 <br> 5. 回到首页 |
| **班组长** | 负责班组生产绩效的初步分析和汇报。 | 1. 登录系统 <br> 2. 进入生产月报表 <br> 3. 点击“上一月”，与上个月的数据进行对比 <br> 4. 点击“趋势”按钮，查看近期几个月的产量变化曲线 <br> 5. 点击“导出”，选择装置，下载报表用于汇报。 |
| **生产主管** | 负责监控整体生产计划完成情况，进行成本和效率分析。 | 1. 登录系统 <br> 2. 进入生产月报表 <br> 3. 浏览所有4个分区（产品、原料、公辅、冷剂）<br> 4. 对比统计卡片的关键指标（如LNG月产量）与预期值 <br> 5. 如发现数据异常（如某指标为负值），会切换到日报表查看每天的具体数据。 |

## 3. 业务流程 (Workflows)

- **主流程 (Happy Path)**: 登录 → 首页 → 生产管理 → 生产月报表 → 页面加载，显示当前月份数据 → 查看统计卡片和表格数据 →（可选）导出/打印 → 退出
- **备选流程-A**: 登录 → 进入月报表 → 点击“上一月/下一月” → 页面切换月份并刷新数据 → 点击“趋势” → 弹窗显示趋势图 → 关闭弹窗 → 继续浏览
- **备选流程-B**: 登录 → 进入月报表 → 发现当前月报表数据未更新（生成报表disabled） → 切换到一个未来月份或已生成报表的月份 → 生成报表按钮启用 → 点击“生成” → 数据刷新
- **异常流程**: 登录 → 进入月报表 → 网络超时 → 页面显示loading或错误提示 → 用户点击“生成报表” →（如果接口未做幂等性）多次点击导致多次请求。

## 5. 数据流 (Data Flow)

- **数据来源**: 当前页面的所有数据（统计卡片、表格行）均通过后端API动态获取，入口URL为 `#/production/monthly-report`。数据来源是生产数据库中的数据表，这些表的数据由`daily-report`（日报表）页面通过“录入/补录/调整”功能写入。
- **数据变换**: 本页面本身**不对数据进行任何写操作**。前端仅负责通过月份参数请求后端接口，并将返回的JSON数据渲染到表格和卡片上。
- **数据消费**:
    - **本页面**: 展示和查看。
    - **其他模块**: 生产月报的数据可能被**报表中心**或其他BI看板系统通过API间接调用，用于生成更高级别的汇总报告（⚠️推断，需确认）。
- **跨页面数据一致性要求**:
    - `daily-report`页面处理完每天的录入数据后，`monthly-report`页面应能准确汇总并展示当月累计值。
    - 例如：日报中1月1日到1月31日的所有“LNG产品”产量之和，应该等于月报表1月份LNG的“产量月累计”值。
