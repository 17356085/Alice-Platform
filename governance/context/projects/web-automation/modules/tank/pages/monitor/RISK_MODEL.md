# 业务场景模型：液罐监控页面 (monitor)

## 1. 业务目标 (Business Goal)

- **核心业务目标**：为罐区操作员提供一个实时、准确、直观的液罐运行状态监控中心，帮助其及时发现并响应异常工况，保障生产安全与效率。 ✅
- **次要业务目标**：为管理人员提供关键指标的宏观视图，辅助生产调度决策。 ⚠️
- **非目标**：本页面不直接控制现场阀门、泵等执行机构，所有控制操作需跳转至专门的“操作控制”页面完成。 ⚠️

## 2. 角色与旅程 (Roles & Journeys)

| 角色 | 职责边界 | 典型旅程 (Journey) |
|------|----------|---------------------|
| **操作员** | 负责日常监控，响应报警，执行常规作业。 | 1. 登录系统进入监控页。 2. 扫描总览仪表盘，检查各罐位液位/温度/压力是否在正常范围。 3. 识别到一个液位偏高的报警。 4. 双击该罐进入“罐详情”弹窗，查看液位变化趋势图。 5. 判断需要排液，点击“排液”按钮（跳转至操作控制页面）。 6. 完成后回到监控页，确认报警已消除。  ✅ |
| **主管 / 调度** | 负责审核操作记录，调整报警阈值，查看全局生产状态。 | 1. 登录系统进入监控页。 2. 切换到“告警历史”或“操作日志”视图，审核过去24小时的关键事件。 3. 点击“配置”按钮（可能需要二次验证），打开报警阈值配置面板。 4. 调整某个罐的液位上限，并保存。 5. 检查保存后，数据模拟能否触发新的报警。  ⚠️ |
| **管理员** | 负责用户权限管理，系统参数配置。 | 旅程主要在“系统管理”模块，监控页面只用于验证权限设置效果。 ❓ |

## 3. 业务流程 (Workflows)

| 流程名称 | 类型 | 流程描述 | 涉及的页面/组件 |
|----------|------|----------|-----------------|
| **正常监控流程** | Happy Path | 操作员登录 → 主页加载 → 监控总览（仪表盘）实时显示数据 → 操作员周期性扫视 → 未发现异常 → 退出系统。 | 监控页 |
| **报警响应流程** | Happy Path | 系统触发报警 → 监控页弹窗/音响提示 → 操作员查看报警列表 → 点击某条报警跳转到罐详情 → 分析趋势图 → 决策并执行排液/升温等操作（跳转操作页面） → 返回确认报警恢复。 | 监控页, 报警组件, 罐详情弹窗, 操作控制页 |
| **报警阈值调整流程** | Alternative Path | 管理员登录 → 监控页 → 点击设置（配置模式） → 选择需要修改阈值的罐 → 输入新阈值（如液位上限从80%调至85%） → 点击保存 → 系统校验合理性（如不超过罐容） → 保存成功。 | 监控页, 配置面板(组件) |
| **数据延迟或断连处理流程** | Exceptional Path | WebSocket连接断开 → 页面显示“通信中断”/“数据更新于...秒前”告警 → 数据停止刷新 → 自动尝试重连 → 重连成功 → 数据恢复更新并提示“已恢复”。 | 监控页, 网络状态组件 |
| **无权限访问流程** | Exceptional Path | 普通操作员通过URL直接访问配置接口 → 后端返回403 → 前端捕获，弹出“无权限”提示。  | 监控页（或任意页面）|

## 4. 业务规则 (Business Rules)

| 规则 ID | 规则描述 | 规则类型 | 置信度 |
|---------|----------|----------|--------|
| RULE-001 | **安全阈值规则**: 液位值不能超过罐容的95%；压力值不能超过设计压力的105%。 | 业务校验规则 | ✅ |
| RULE-002 | **报警产生规则**: 当液位超过安全上限(95%)时，自动生成并推送 红色（最高级）报警；当液位从低位恢复至正常范围(70%)以下，报警自动消除。 | 状态流转 / 触发规则 | ⚠️ |
| RULE-003 | **数据更新规则**: 前端每隔2秒向后端拉取最新数据；后端应保证推送的数据时间戳误差不超过1秒。 | 实时性计算规则 | ❓ |
| RULE-004 | **权限校验规则**: 只有角色为“主管”或“管理员”的用户才能访问“报警阈值配置”的API和数据。 | 权限规则 | ✅ |

## 5. 数据流 (Data Flow)

- **数据来源**: 后端WebSocket服务通过`/ws/monitor/tank/{tankId}`推送实时数据。历史数据通过常规REST API `/api/v1/tank/{tankId}/records?from=...&to=...` 获取。 ✅
- **数据变换**: 前端对原始JSON数据进行解析，将数值（如水位高度 cm）转换为百分比（液位 %）；将时间戳转换为可视化格式（如 `HH:mm:ss`）。 ✅
- **数据消费**: 本页面的数据直接渲染为仪表盘、实时表格和趋势图。操作员基于这些数据在“操作控制”页面(`operation`)执行排液、加温等操作。报警数据会写入后端数据库，并在“告警管理”页面(`alarm-management`)进行归档和查询。 ✅
- **跨页面一致性要求**: 在“操作控制”页面执行的“排液”操作后，液位应在2-3秒内在“监控”页面得到同步更新。 ⚠️
