## 模块定位

库管管理是鞍集涂源管理系统的**物料出入库中枢**，覆盖备品备件和环保危废两大类物料的：
- 入库/出库操作
- 库存查询与盘点
- 领用申请流程
- 物品主数据管理
- 装填作业管理（三剂消耗管理）

**区别于其他模块：** 库管管理侧重"物料流动"，与设备管理（侧重"设备台账"）、生产管理（侧重"生产执行"）互补。

## 🔗 跨模块关联链路

### 审批流程全链路

```
库管管理                              工作流管理 (workflow)
─────────                            ────────────────────
环保出库/环保入库                      待我审批 / 我已审批 / 我发起的 / 审批链配置
     │                                      ↑
     │  ① 新增出库/入库单                     │
     │  ② 点击"提交"                         │
     │  ③ ──── 工作流引擎路由 ────→           │  ④ 审批人看到待审批任务
     │                                      │  ⑤ 审批通过 / 审批驳回
     │  ⑥ ←─── 审批结果回写 ←────────        │
     │  ⑦ 通过→出/入库生效                    │
     │     驳回→回到草稿                      │
```

## 核心业务规则

### 出入库规则（推测）
1. 入库操作 → 库存数量增加；出库操作 → 库存数量减少
2. 库存数量不能为负数（出库时校验库存充足）
3. 备品备件和环保危废的库存独立管理（两套库存账）

### 审批流状态机（环保出库/入库）
```
草稿 ──提交──→ 审批中 ──审批通过──→ 已生效
                 │
                 └──审批驳回──→ 已驳回（可重新编辑提交）
```

### 物品管理规则
1. 物品管理维护物料主数据（编码、名称、规格、单位等）
2. 入库/出库时从物品主数据中选择物料
3. 物品管理支持导入导出（批量操作）

### 盘点规则
1. 库存盘点对比账面数与实盘数
2. 盘点差异通过"盘点调整"处理

## 自动化实施优先级

基于业务重要性和复杂度，建议分 3 批推进：

| 批次 | 页面 | 理由 |
|:----:|------|------|
| **P0（首批）** | 环保入库、环保出库 | 审批流核心链路，复杂度最高，bug风险最大 |
| **P1（二批）** | 备品入库、备品出库、备品领用申请、备品库存查询 | 高频操作，业务主流程 |
| **P2（三批）** | 备品出入库记录、备品库存盘点、盘点调整、物品管理×3、装填管理 | 辅助功能/低频操作 |
| **P3（末批）** | 数字化大屏 | 纯展示，低交互 |
