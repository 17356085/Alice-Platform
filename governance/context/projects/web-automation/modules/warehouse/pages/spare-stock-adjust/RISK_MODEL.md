## 7. 业务场景

### 角色与旅程

| 角色ID | 角色名称 | 职责边界 | 典型旅程 | 置信度 |
|--------|----------|----------|----------|--------|
| ROLE-SA-001 | 仓管员 | 负责备品备件的盘点调整操作，并通过本页面核对调整记录 | 1. 登录系统 → 2. 导航至页面 → 3. 输入物品编号查询特定调整记录 → 4. 核对调整前后的数量是否合理 → 5. 关闭页面 | 从 PO 代码推断 |
| ROLE-SA-002 | 库管经理 | 负责仓库整体管理，关注盘点调整的汇总数据 | 1. 登录系统 → 2. 导航至页面 → 3. 选择日期查看当天的所有调整记录 → 4. 通过分页浏览全量记录 → 5. 关闭页面 | 从 PO 代码推断 |
| ROLE-SA-003 | 审计人员 | 对备品盘点调整的合规性进行检查 | 1. 登录系统 → 2. 导航至页面 → 3. 根据时间范围逐页核对调整记录 → 4. 确认调整审批流程已执行 | 从业务场景推断 |

### 业务流程

| 流程ID | 流程类型 | 流程描述 | 涉及页面 | 关联角色 | 置信度 |
