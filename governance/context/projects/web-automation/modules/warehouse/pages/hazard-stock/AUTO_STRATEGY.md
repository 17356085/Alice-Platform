# 自动化策略: 环保库存查询 (Hazard Stock)

## 1. 自动化覆盖矩阵

| 用例编号 | 标题 | 优先级 | 是否自动化 | 理由 |
|---|---|---|---|---|
| TC-HS-001 | 页面正常加载 — 表格渲染正常 | P0 | ✅ 已自动化 | 基础冒烟，验证页面核心组件（表格、分页）渲染正常。 |
| TC-HS-002 | 分页组件可见性验证 | P0 | ✅ 已自动化 | 验证列表页关键组件可用性。 |
| TC-HS-003 | 按危废品名称搜索 | P1 | ✅ 已自动化 | 核心查询功能，验证搜索流程不报错。搜索"test"验证输入框和查询按钮交互。 |
| TC-HS-004 | 重置搜索条件 | P1 | ✅ 已自动化 | 核心UI功能，验证重置行为正常。 |
| TC-HS-005 | 搜索无结果时的空数据展示 | P2 | ❌ 未自动化 | 非冒烟场景，当前自动化脚本未覆盖。可后续扩展。 |

**覆盖率总结**: 核心 P0/P1 用例已全部自动化（4/5 用例），覆盖率达 80%。未覆盖的 P2 级用例影响有限。

## 2. PageObject 拆分方案

| Page 类 | 职责 | 关键方法 |
|---|---|---|
| **HazardStockPage** | 主列表页的搜索/重置/表格/分页操作，所有交互均在主页面完成。 | `search_by_item_name`, `reset_search`, `navigate` |

**决策理由**: 页面交互极简（仅搜索和重置），无弹窗/无 CRUD，无需拆分。单个 Page 类完全胜任。

## 3. 公共组件复用分析

| 公共能力 | 复用情况 | 说明 |
|---|---|---|
| `BasePage.navigate_to` | ✅ **直接复用** | 三参数导航：库管管理 → 环保危废管理 → 库存查询 |
| `BasePage._wait_loading_gone` | ✅ **直接复用** | 在 `_wait_page_ready` 中调用，等待 Element Plus 加载动画消失 |
| `BasePage.wait_vue_stable` | ✅ **直接复用** | 搜索、重置后调用，确保 Vue 响应完成 |
| `BasePage.input_text` | ✅ **直接复用** | 输入搜索关键词 |
| `BasePage.click` | ✅ **直接复用** | 点击查询/重置按钮 |
| `BasePage.TABLE_ROWS` | ✅ **直接复用** | 获取表格行数 |
| `BasePage.TOTAL_COUNT` | ✅ **直接复用** | 获取分页总数 |

**建议**: 当前 BasePage 能力已完全满足该页面的自动化需求，无需扩展 ElementPlusHelper。

## 4. 等待策略建议

| 场景 | 当前策略 | 建议 | 原因 |
|---|---|---|---|
| 页面初始加载 | `_wait_loading_gone()` + `wait_vue_stable()` | 保持当前策略 | 最可靠的组合 |
| 搜索/重置后 | `wait_vue_stable()` | 保持当前策略 | 确保 Vue 响应完成后进一步操作 |

## 5. ROI 分析

| 项目 | 评估值 | 说明 |
|---|---|---|
| 手工执行时间 (每次) | ~2 分钟 | 包括登录、导航、输入搜索、重置等简单操作。 |
| 预估开发成本 | **2 小时** | 页面极简，仅 3 个定位器 + 3 个方法，测试 4 个用例。 |
| 预估维护成本 | **0.5 小时/月** | 定位器数量少，仅搜索/重置按钮文本变化可能引发维护。 |
| 执行频率 (假设) | 每日 2 次 (CI) | 在 CI/CD 中作为回归测试的一部分执行。 |
| 月度手工执行成本 | 2 分钟 x 2 次 x 30 天 = 120 分钟 = **2 小时** | 每月手工执行时间。 |
| 月度自动化维护成本 | **0.5 小时** | 维护成本极低。 |
| ROI 计算 (首月) | 手工成本(2h) - 开发成本(2h) - 维护成本(0.5h) = **-0.5h** | 首月接近收支平衡。 |
| ROI 计算 (次月及以后) | 手工成本(2h) - 维护成本(0.5h) = **1.5h** | 次月开始每月净节约 1.5 小时。 |

**结论**: 投入极低，回报周期短。该页面自动化是高 ROI 的典型场景。
