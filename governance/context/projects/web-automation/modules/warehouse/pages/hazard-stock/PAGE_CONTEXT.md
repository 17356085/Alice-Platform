# 页面上下文: 环保库存查询 (Hazard Stock)

| 元数据项 | 值 |
|---|---|
| 模块 | warehouse (库管管理) |
| 页面名称 | 环保库存查询 |
| 访问路径 | 库管管理 → 环保危废管理 → 库存查询 |
| Hash 路由 | `#/warehouse/hazard/stock` (从 conftest.py 推断) |
| 页面类型 | 只读查询列表页 |
| 主要功能 | 查看危废品库存、按危废品名称搜索、重置搜索条件 |
| 审批流 | 无 |
| 数据操作 | 无 CRUD，仅查询展示 |

## 1. 页面整体结构

该页面为只读查询列表页，主要分为以下几个区域：

1. **顶部/导航**: 页面标题和面包屑导航（由框架提供，非本页特有）。
2. **搜索/筛选区 (Filter Area)**: 位于页面顶部，用于按条件筛选库存数据。
   - 包含：危废品名称输入框、查询按钮、重置按钮。
3. **表格/列表区 (Table Area)**: 用于展示危废品库存数据列表。
   - 包含：多选框列（从方法推测）、危废品名称列、库存数量列、其他数据列。
4. **分页区 (Pagination Area)**: 位于表格底部，用于翻页和数据量统计。

## 3. 页面状态

| 状态 | 表现 | 交互/等待策略 |
|---|---|---|
| 加载中 | 表格区域出现 `el-loading-mask` 覆盖层 | 使用 `_wait_loading_gone()` 等待加载动画消失 |
| 空数据 | 表格区域显示空数据提示 | 搜索后应断言数据行数，或获取总数是否为0 |
| 错误状态 | 可能由后端接口返回，前端展示错误提示 | 依赖于 `el-message` 组件，由 BasePage 通用方法处理 |

## 4. 权限点

根据测试脚本，未发现明显的权限控制点。所有测试用例均基于一个拥有完整访问权限的用户执行。查询按钮对任何有页面访问权限的用户均可见。如果未来引入权限控制，页面菜单入口将是潜在的权限控制点。
