## 2. PageObject 拆分方案

根据"一个页面一个Page类"的原则，建议如下拆分：

```
page_objects/
└── warehouse/
    └── spare_stocktake/
        ├── __init__.py
        └── spare_stocktake_page.py     # 主页面类（无弹窗，单一文件）
```

### `SpareStocktakePage`
- **职责**: 封装库存盘点页面的搜索、表格、分页操作。
- **核心方法**:
  - `search_by_handler(handler_name: str)` -> `self`
  - `search_by_date(date_range: tuple)` -> `self` (可选，需扩展)
  - `click_reset()` -> `self` (或 `reset_search()`)
  - `get_total_count()` -> `int`
  - `check_page_loaded()` -> `bool`
- **不包含**: 无 CRUD 方法，无弹窗处理。

## 3. 公共组件复用分析

### 3.1 复用 `BasePage` 已有方法（高频）

- **`wait_for_element_visible(locator)`**: 搜索框、按钮、表格交互前调用。
- **`click_element(locator)`**: 点击查询、重置按钮。
- **`type_text(locator, value)`**: 搜索区的盘点人输入框。
- **`get_element_text(locator)`**: 获取分页总条数。
- **`take_screenshot(name)`**: 测试失败时截图。

### 3.2 复用 `ElementPlusHelper`（需要考虑）

- **`select_date(locator, date_str)`**: 日期选择器组件（`el-date-picker`）。如果当前 `ElementPlusHelper` 已封装日期选择方法，可直接复用。否则需要扩展。

### 3.3 需要扩展 `ElementPlusHelper` 或新建工具方法

- **`handle_date_picker_range(locator, start_date, end_date)`**: 针对 `el-date-picker` 范围选择模式，可能需要封装选择开始日期和结束日期的方法。
- 上述方法为可选项，如果当前 PO 中未实现日期筛选的自动化测试，可不实现。

## 4. 等待策略建议

该页面特有的异步行为建议如下处理：

1. **表格加载**:
   - 行为: 搜索/重置/分页切换后，表格数据异步请求并渲染。
   - 策略: 等待 `el-table` 中的 loading 遮罩消失。

2. **日期选择器打开**:
   - 行为: 点击日期输入框后，日历面板异步渲染到 `<body>` 下。
   - 策略: 等待 `.el-picker-panel` 或对应日历可见。

3. **搜索**:
   - 行为: 输入盘点人后点击查询，页面异步请求。
   - 策略: 输入内容后点击查询按钮，等待 loading 消失。

4. **重置**:
   - 行为: 点击重置后，搜索框内容清空，日期选择器重置，表格恢复初始数据。
   - 策略: 点击重置按钮，等待 loading 消失。

```python
# 建议的等待封装结构
def wait_for_table_ready(page, timeout=10):
    """等待表格数据加载完毕"""
    page.wait_for_element_invisible(by_css='.el-loading-mask')
```

## 5. ROI 分析

### 假设条件

- 预期执行频率: **每日 3 次** (CI/CD构建)。
- 手工执行时间: 完成页面加载、分页、搜索、重置、日期筛选测试约为 **10 分钟/次**。
- 预估自动化开发时间: **2.5 小时** (仅 `SpareStocktakePage` 和 4 个测试方法，加上日期选择器封装）。
- 预估月度维护成本（按平均每月页面变动2次计算）：**0.4 小时/月**。

### ROI 计算

- **月度手工执行时间**: `10 分钟/次 × 3 次/天 × 22 工作日/月` = **660 分钟/月** ≈ **11 小时/月**
- **月度自动化执行时间**: 假设自动化脚本执行耗时 1.5 分钟（含日期选择器），则 `1.5 分钟/次 × 3 次/天 × 22 天/月` = **99 分钟/月** ≈ **1.65 小时/月**
- **月度节省时间**: `11 - 1.65 = 9.35 小时/月`
- **月度净收益**: `月度节省时间 - 月度维护成本` = `9.35 - 0.4` = **8.95 小时/月**
- **投资回收期**: `总投资（开发时间） / 月度净收益` = `2.5 / 8.95` ≈ **0.28 个月 → 约6个工作日**

**结论**: 自动化测试的投资回报率（ROI）较高。初始开发的 **2.5 小时** 成本，在 **约6个工作日** 内即可收回，后续每月可释放约 **8.95 小时** 的人力资源。
