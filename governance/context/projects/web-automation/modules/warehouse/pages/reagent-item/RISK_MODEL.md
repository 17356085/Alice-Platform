## 业务场景

### 角色与旅程

| 角色 | 职责边界 | 典型操作 |
|---|---|---|
| 库管员 | 日常维护三剂物品目录，确保目录准确可用 | 查看、搜索、新增、删除 |
| 三剂管理员 | 管理三剂消耗物品，关注消耗统计 | 查看、搜索、新增 |
| 系统管理员 | 配置权限，监控系统使用 | 所有操作 |

### 典型业务旅程

**库管员 - "新增一种三剂物品" 旅程**
1. **进入**: 导航到 `库管管理 > 三剂消耗管理 > 物品管理`，页面加载并展示当前物品列表。
2. **确认**: 搜索检查系统中是否已存在同名物品。如果存在则复用，流程结束。
3. **新建**: 点击"新增"按钮，弹出新增表单弹窗。
4. **填写**: 在弹窗中填写物品名称（如 AUTO_三剂_001）。
5. **保存**: 点击"保存"按钮，表单提交，弹窗关闭。
6. **验证**: 搜索确认新物品已在列表中。旅程结束。

**库管员 - "删除废弃物品" 旅程**
1. **搜索**: 输入物品名称搜索定位到目标记录。
2. **触发删除**: 在该行点击"删除"按钮。
3. **确认**: 系统弹出确认弹窗，点击"确定"。
4. **验证**: 系统提示删除成功，列表中物品消失。旅程结束。

### 业务流程

- **Happy Path**: 新增物品 → 搜索确认 → 删除清理
- **Alternative Path 1 (取消新增)**: 点击新增 → 填写信息 → 点击取消 → 未创建数据
- **Alternative Path 2 (必填校验)**: 点击新增 → 不填名称点保存 → 显示校验错误
- **Exception Path (删除失败)**: 点击删除 → 后端返回错误（物品已关联消耗记录） → 显示错误提示
