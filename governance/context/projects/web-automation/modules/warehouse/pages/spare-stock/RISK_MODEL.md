# 风险模型: warehouse → spare-stock

## 页面概述
- **页面名称**: 备品库存查询 (Spare Stock Query Page)
- **模块**: 仓库管理 (Warehouse)
- **核心功能**: 展示备品库存信息，支持按物品名称搜索查看库存量。只读查询页面，无 CRUD 操作。

## 风险识别

### 1. 业务风险 (Business Risks)

| 风险ID | 风险描述 | 等级 | 缓解措施 |
|--------|----------|------|----------|
| RISK-SS-001 | **库存数据展示不及时**: 库存查询页面展示的数据未与实际出入库操作实时同步，导致业务决策失误（如判断库存充足实际缺料）。 | P1 | 1. 页面数据应设置合理的刷新机制（触发式刷新）。<br>2. 数据展示区增加"最后更新时间"标签。<br>3. 重要操作后强制刷新列表。 |
| RISK-SS-002 | **库位信息不准确导致领料失败**: 展示的库存库位与物理位置不符，导致无法找到物料。 | P1 | 1. 数据源必须实时同步 WMS 系统。<br>2. 页面提供库位变更记录查询入口。 |
| RISK-SS-003 | **搜索不准确导致库存误判**: 模糊搜索匹配不完整或精确搜索无结果，用户误判为无库存。 | P2 | 1. 支持模糊搜索和精确搜索两种模式。<br>2. 无结果时提供"修改搜索条件"的引导提示。 |

### 2. 权限风险 (Permission Risks)

| 风险ID | 风险描述 | 等级 | 缓解措施 |
|--------|----------|------|----------|
| RISK-SS-004 | **越权查看库存数据**: 普通用户通过 URL 直接访问库存查询页面，看到不应看到的库存信息。 | P0 | 1. 后端 API 必须校验用户对库存数据的访问权限。<br>2. 不能仅依赖前端菜单隐藏来控制访问。 |
| RISK-SS-005 | **数据范围越权**: 某仓库管理员只能查看自己仓库的库存，但页面显示了全部仓库数据。 | P0 | 1. 后端 API 根据用户角色和组织范围过滤数据。<br>2. 页面应默认加载用户有权限的数据范围。 |

### 3. 数据风险 (Data Risks)

| 风险ID | 风险描述 | 等级 | 缓解措施 |
|--------|----------|------|----------|
| RISK-SS-006 | **搜索框输入注入攻击**: 搜索框输入特殊字符（如 SQL 注入、XSS 攻击向量）导致安全问题。 | P0 | 1. 所有搜索输入参数使用参数化查询。<br>2. 对前端展示的备件名称等字段进行 HTML 转义。 |
| RISK-SS-007 | **库存数量显示异常**: 库存量为极大值、负值或 NaN 时，页面显示异常（如科学计数法、格式错乱）。 | P1 | 1. 前端对数字字段进行格式化处理，处理边界值。<br>2. 后端对存储值进行业务校验。 |
| RISK-SS-008 | **空列表展示不友好**: 搜索无匹配结果后，页面仅显示空白区域，未提供引导提示。 | P2 | 提供明确的空状态提示和操作建议。 |

### 4. 接口风险 (Interface Risks)

| 风险ID | 风险描述 | 等级 | 缓解措施 |
|--------|----------|------|----------|
| RISK-SS-009 | **接口超时加载态不清晰**: 请求列表数据时接口响应慢，页面一直显示 Loading 遮罩，用户无法操作。 | P1 | 1. 设置前端请求超时时间，超时后展示重试按钮。<br>2. 请求期间阻止用户反复点击搜索按钮。 |
| RISK-SS-010 | **分页接口返回异常**: 请求第 N 页，后端返回了错误页的数据。 | P1 | 前端对返回的分页数据结构进行校验，`total` 与 `list.length` 一致性检查。 |
| RISK-SS-011 | **Token 过期无感知**: Token 过期后接口返回 401，页面无跳转或无明确提示。 | P0 | 前端全局拦截器捕获 401 状态码，自动跳转至登录页或弹出"会话过期"提示。 |

### 5. UI/UX 风险 (UI/UX Risks)

| 风险ID | 风险描述 | 等级 | 缓解措施 |
|--------|----------|------|----------|
| RISK-SS-012 | **表格固定列重叠**: 小屏幕分辨率下 `el-table` 固定列与普通列重叠。 | P1 | 1. 进行多分辨率（1366、1920）下的布局测试。<br>2. 超出时提供横向滚动。 |
| RISK-SS-013 | **搜索框无防抖导致频繁请求**: 用户快速输入时每次 keystroke 都发起请求（如果实现了即时搜索），导致前端卡顿。 | P2 | 使用防抖（debounce）技术，建议 300ms。 |

### 6. 性能风险 (Performance Risks)

| 风险ID | 风险描述 | 等级 | 缓解措施 |
|--------|----------|------|----------|
| RISK-SS-014 | **列表首次加载数据量大**: 一次性加载大量库存记录导致渲染卡顿。 | P0 | 1. 必须实现后端分页。<br>2. 默认显示前 20 条，用户翻页加载更多。 |
| RISK-SS-015 | **频繁搜索导致服务器压力**: 用户在短时间内多次搜索不同物品名称，导致后端接口高并发。 | P2 | 使用防抖或节流技术限制请求频率。 |

## 业务场景

### 1. 角色与旅程

| 角色 | 职责边界 | 典型旅程 | 置信度 |
|------|----------|----------|--------|
| **仓库管理员** | 查看库存情况，进行库存盘点、调拨。 | 打开页面 → 搜索物品名称 → 查看当前库存量 → 决定是否需要补货或调拨。 | ⚠️ 推断 |
| **采购专员** | 查询备品库存，制定采购计划。 | 打开页面 → 搜索物品名称 → 查看库存量 → 导出或记录后创建采购订单。 | ⚠️ 推断 |
| **生产操作员** | 查看需领用的物料是否有库存。 | 打开页面 → 搜索物料名称 → 确认有库存 → 前往领料。 | ⚠️ 推断 |

### 2. 业务流程

- **Happy Path**: 仓库管理员登录 → 导航至库存查询 → 输入物品名称 → 点击查询 → 查看搜索结果中的库存量。
- **Alternative Path 1**: 用户搜索不存在的物品 → 页面展示空状态 → 用户修改搜索条件 → 再次查询。
- **Alternative Path 2**: 用户点击重置 → 搜索条件清空 → 表格恢复为全部库存数据。

### 3. 业务规则

| 规则ID | 规则描述 | 验证点 | 置信度 |
|--------|----------|--------|--------|
| BR-SS-001 | **库存量实时性**: 库存查询数据应与最新出入库记录保持最终一致性。 | 在出入库操作后，验证库存量更新。 | ⚠️ 推断 |
| BR-SS-002 | **数据范围控制**: 用户只能查看其有权限的仓库/组织的数据。 | 使用不同角色登录，查看列表数据范围。 | ⚠️ 推断 |

## 未覆盖的高风险场景（自动化用例缺失）
- **RISK-SS-004 / RISK-SS-005**: 缺少权限校验的自动化测试（越权访问、数据范围）。
- **RISK-SS-006**: 缺少对搜索框进行 XSS/SQL 注入攻击的接口安全测试。
- **RISK-SS-014**: 缺少对大数据量下的渲染性能自动化验证。
