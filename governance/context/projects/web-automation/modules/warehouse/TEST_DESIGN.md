## 数据策略

| 维度 | 内容 |
|------|------|
| 测试数据来源 | 系统现有数据 + 自动化脚本生成（按"自动化测试-"前缀标识） |
| 数据清理方式 | fixture teardown 中按前缀删除测试创建的入库/出库单据 |
| 前置准备 | ①确保测试账号已分配对应角色 ②确认危废品物品管理中有可用数据 ③确认 chenqian 和仓管审批人账号可用 |
| 数据隔离 | 自动化测试数据与手工测试数据通过命名前缀隔离 |
| 审批数据 | 端到端审批需要 3 个账号：提交人(admin)、主管审批(chenqian)、仓管审批人(待确认) |

## 自动化建议

| 建议 | 场景数 | 说明 |
|------|:------:|------|
| ✅ 可自动化 | ~35 | 元素交互、搜索/分页、CRUD、弹窗操作、权限校验（占80%+） |
