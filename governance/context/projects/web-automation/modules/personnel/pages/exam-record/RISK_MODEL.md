## 1. 业务目标 (Business Goal)

- **核心业务目标**: 追溯和查看特定员工的历史考核/考试情况，用于绩效考核、资质认证或合规审查。
- **次要业务目标**: 批量导出考试记录数据，供线下统计分析或存档。
- **非目标**: 本页面不创建、编辑或删除考试记录本身，它只是一个只读的历史数据展示和查询工具。

## 2. 角色与旅程 (Roles & Journeys)

| 角色 | 职责边界 | 典型旅程 |
| :--- | :--- | :--- |
| **系统管理员 (Admin)** | 查看**所有**员工的考试记录，具备所有操作权限。 | 登录 → 导航到“考试记录” → 输入员工姓名查询 → 点击查看某条详情 → 导出月度考试汇总表。 |

| **培训专员 (Trainer)** | 查看所有记录，但可能无导出权限。 | 登录 → 导航到“考试记录” → 按“考试类型”过滤 → 查看某个人员考试详情 → 判断是否需要补考。 |

## 3. 业务流程 (Workflows)

**Happy Path (查看详情)**:
1. 用户在搜索框输入“张三”。

3. 点击“搜索”按钮。
4. 表格刷新，显示张三在此期间的考试记录。
5. 用户点击某行记录的“查看”按钮。
6. 系统弹出“考试记录详情”弹窗，展示完整信息。
7. 用户点击“确定”关闭弹窗。

**Alternative Path 1 (导出失败)**:
1. 用户选择所有查询条件后，点击“导出”按钮（用户有导出权限）。
2. 按钮变为“导出中...”（加载中状态），按钮置灰。
3. `⚠️ 推断`: 系统提示“导出失败，请稍后重试！”或成功导出文件。
4. 若失败，用户关闭提示，稍后再试。
