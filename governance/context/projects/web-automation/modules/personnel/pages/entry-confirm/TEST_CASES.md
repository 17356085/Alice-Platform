## TC-EC-010: 单条确认入场
- **映射场景**: TD-EC-010
- **前置条件**: 存在状态为"未读"的记录
- **步骤**:
  1. 找到第一条"未读"状态记录
  2. 点击行内"确认入场"按钮
  3. 若有确认弹窗，点击确定
  4. 等待 toast
- **断言**:
  - toast 提示"确认成功"
  - ⚠️ 该记录状态变更（不可逆）

## TC-EC-011: 批量确认入场
- **映射场景**: TD-EC-011
- **前置条件**: 存在 ≥2 条"未读"记录
- **步骤**:
  1. 勾选 ≥2 行复选框
  2. 点击"批量确认入场"按钮
  3. 若有确认弹窗，点击确定
  4. 等待 toast
- **断言**:
  - toast 提示成功
  - ⚠️ 所选记录批量确认（不可逆）

- **映射场景**: TD-EC-012
