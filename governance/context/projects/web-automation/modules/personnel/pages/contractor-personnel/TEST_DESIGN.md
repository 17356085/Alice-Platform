# TEST_DESIGN — personnel / contractor-personnel

## 基本信息
- 模块：personnel
- 页面：承包商人员
- 测试目标：验证承包商人员的 CRUD 全流程、搜索筛选、侧边栏导航切换及异常处理
- 需求来源：PAGE_CONTEXT.md + 线上 DOM 诊断（2026-06-15）

## 测试范围
- 包含：页面加载（含侧边栏视图切换）、搜索（按姓名/身份证/单位/状态）、分页、新增、编辑、启停用、删除
- 不包含：权限校验、批量导入

## 特殊说明
- **无独立路由**：与承包商单位共用 `#/personnel/contractor`，导航需两步：hash 跳转 + 侧边栏点击 nest-menu 项
- 侧边栏导航前提：父级 "人员管理" 和 "承包商管理" sub-menu 已展开

## 数据策略
- 测试数据来源：运行时生成唯一姓名（`测试人员_{timestamp}`）
- 数据清理方式：`conftest._teardown_contractor_personnel()` 按姓名前缀搜索删除
- 前置准备：需先存在承包商单位（选择所属单位时使用）

## 自动化建议
- 建议自动化：搜索/CRUD/分页 — P0 100%，P1 100%
- 注意：侧边栏导航依赖 sub-menu 展开状态，定位器需处理已展开/未展开两种情况
- 定位器稳定性：card 搜索区使用 `.search-wrapper` class 定位
