# RISK_MODEL — workflow（工作流管理 5 页面）

> 基于 3 轮 Bug 分析的实测发现 | 2026-06-15
> 从 system/RISK_MODEL.md 提取工作流相关风险

## 风险清单

### 1. 业务风险

| 风险ID | 描述 | 影响 | 等级 | 缓解措施 |
|:---:| --- | --- |:---:| --- |
| RM-WF-001 | **审批链配置误删**：删除审批链后，依赖该审批链的业务流程（如日报审批）无法正常流转 | 审批流程中断 | **P0** | 1) 删除前校验是否有关联业务；2) 软删除+恢复机制；3) 操作日志 |
| RM-WF-002 | **审批操作不可逆**：通过/驳回审批后无法撤销，误操作导致数据错误 | 业务数据异常 | **P1** | 1) 审批确认弹窗+意见必填；2) 操作后展示结果摘要 |
| RM-WF-003 | **审批链编码重复**：编码是SAP集成的唯一标识，重复会导致推送错误 | SAP推送失败 | **P0** | 1) 后端唯一性校验；2) 新增/编辑时异步检查 |

### 2. 权限风险

| 风险ID | 描述 | 等级 | 缓解措施 |
|:---:| --- |:---:| --- |
| RM-WF-004 | **越权审批**：用户可审批不属于自己权限范围内的申请 | **P0** | 1) 后端校验审批人身份；2) 审批链配置中限定审批角色 |
| RM-WF-005 | **SAP推送日志泄露**：日志中可能包含敏感业务数据（工厂产量等），不应向低权限用户开放 | **P1** | 1) 日志脱敏；2) 按角色限制可见数据范围 |

### 3. 数据风险

| 风险ID | 描述 | 等级 | 缓解措施 |
|:---:| --- |:---:| --- |
| RM-WF-007 | **工作流搜索字段非标准**：工作流页面用"工厂代码"而非"标题"，与其他CRUD页面不一致，自动化代码易误写 | **P2** | 1) PAGE_CONTEXT 明确标注；2) PO 方法命名体现实际字段名 |
| RM-WF-008 | **审批链表单空名称提交**：弹窗中名称为空时提交，后端应校验但前端提示可能不明确 | **P1** | 1) 前端必填校验+红色提示；2) 后端返回具体字段错误 |
| RM-WF-009 | **空状态文案不统一**：各页面空状态显示"暂无待审批项"/"暂无已审批项"/"暂无数据"，自动化断言需适配 | **P2** | 1) 统一空状态文案；2) 断言用前缀匹配 `"暂无" in text` |

### 5. UI/UX 风险

| 风险ID | 描述 | 等级 | 缓解措施 |
|:---:| --- |:---:| --- |
| RM-WF-013 | **按钮文字在 `<button>` 直贴 vs `<span>` 包裹**：不同页面/组件不一致，XPath `//button[.//span[contains(text(),"X")]]` 不可靠 | **P1** | 1) 统一用 JS textContent 文本搜索；2) 代码红线补充此条 |
| RM-WF-014 | **多 overlay 导致弹窗定位错乱**：页面常驻多个隐藏 overlay（`display:none` CSS），XPath `[last()]` 选错 | **P1** | 1) 统一用 CSS Selector `:not([style*="display: none"])`；2) dialog 检测优先用 CSS |
| RM-WF-015 | **el-dialog vs el-drawer vs el-modal-dialog**：新增/编辑弹窗可能使用不同容器类名 | **P2** | 1) 弹窗定位器覆盖三种容器；2) 诊断脚本先确认实际类名 |
| RM-WF-016 | **el-select 搜索字段无 text input**：工作流搜索区的"工厂代码"是 `<el-select>`（下拉选择），不是 `<el-input>`（文本输入），导致 `send_keys` 失败 | **P1** | 1) PAGE_CONTEXT 标注字段类型；2) PO 方法区分 input 和 select 操作 |
