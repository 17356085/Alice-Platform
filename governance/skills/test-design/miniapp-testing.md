# Skill: miniapp-testing

## 目标
为微信小程序页面设计测试用例，特别关注小程序特有的测试维度。

## 输入
- 小程序页面截图
- 用户操作手册
- 页面名称、所属 Tab

## 输出
- 小程序测试用例
- 小程序特有风险点清单

## 规则
- 覆盖小程序特有维度：微信登录态、网络切换（WiFi ↔ 4G/5G）、前后台切换
- 覆盖授权场景：位置、相机、存储等权限弹窗
- 覆盖兼容性：不同微信版本、不同机型
- 用例格式与 Web 端保持一致

## 依赖
- skills/testcase-design.md
- workflows/sop-summary.md (§ S-06)

## 边界
- 本 Skill 不处理 Web 管理端测试
- 不处理微信支付场景（如涉及需额外补充）

你是一个微信小程序测试工程师。请为以下小程序页面设计测试用例。

## 页面信息

- 所属Tab：{{首页 → 人员培训 → 在线学习}}

## 小程序特有测试点
请特别关注：
- 微信登录态（一键登录/手机号登录/多角色切换）
- 不同微信版本兼容性
- 网络切换（WiFi ↔ 4G/5G）
- 小程序前后台切换（数据恢复/状态保持）
- 分享/转发功能
- 缓存策略（本地数据 vs 服务端数据）
- 授权弹窗（位置/相机/存储等权限）
- 页面栈深度（超过10层后跳转行为）

### 小程序自动化可行性分析

```text
分析以下小程序页面的自动化测试可行性。

## 页面信息
- 页面名称：{{在线学习-课程列表}}
- 技术栈：微信原生小程序 / uni-app / Taro
- 自动化工具：mp-weixin-automator（Node.js）

## 任务
1. **页面结构分析**：
   - 组件类型（swiper/scroll-view/picker/modal等）
   - 是否使用自定义组件（路径、是否有 id/class）
   - 是否有 canvas/web-view（不可自动化区域）

2. **自动化可行性评估**：
   | 功能点 | 可行性 | 定位方式 | 风险 |
   |--------|--------|----------|------|
   | 课程列表滚动 | ✅ 可自动化 | scroll-view > view:nth-child | 虚拟列表场景不可靠 |
   | 微信支付按钮 | ❌ 不可自动化 | — | 需要微信支付密码/指纹 |
   | 分享按钮 | ⚠️ 部分可 | button[open-type="share"] | 只能点击，无法验证分享结果 |
   | 授权弹窗 | ⚠️ 部分可 | wx-authorize组件 | 真机上可能需要真实授权 |

3. **自动化覆盖建议**：
   - P0 必自动化：{{列表展示、搜索、课程详情跳转}}
   - P1 建议自动化：{{视频播放、进度同步}}
   - P2 手工测试：{{支付、分享、消息推送}}

4. **与 Web 端联动测试建议**：
   - 小程序操作 → Web端验证（如：小程序提交培训申请 → Web端审批）
   - Web端操作 → 小程序验证（如：Web端发布课程 → 小程序端可见）
```

- [ ] 覆盖兼容性：不同微信版本/不同机型/iOS vs Android
