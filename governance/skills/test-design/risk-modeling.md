# Skill: risk-modeling

## 目标
从页面上下文出发，识别业务风险、权限风险、数据风险、接口风险等，输出页面级风险模型。

## 输入
- PAGE_CONTEXT.md
- MODULE_CONTEXT.md
- 历史缺陷记录
- 权限矩阵（如有）

## 输出
- RISK_MODEL.md

## 规则
- 从 6 个维度识别风险：业务、权限、数据、接口、UI/UX、性能
- 风险等级：P0（阻塞上线）/ P1（影响核心体验）/ P2（边缘场景）
- 每个 P0 风险必须给出缓解措施
- 关联现有自动化用例覆盖率，标注未覆盖的高风险场景

## 依赖
- templates/risk-model.template.md
- skills/page-analysis.md

## 边界
- 本 Skill 不产出测试设计（那是 testcase-design 的职责）
- 本 Skill 不产出自动化策略（那是 auto-strategy 的职责）

## Prompt 模板

> 以下 Prompt 可直接复制到 AI 对话中使用。替换 `{{ }}` 占位符即可。

### 页面级风险建模

```text
基于以下页面上下文，进行全面的风险建模。

## 页面信息
- 页面：{{用户列表页}}
- 模块：{{系统管理-用户管理}}
- PAGE_CONTEXT：{{粘贴 PAGE_CONTEXT.md 核心内容}}

## 任务
从以下维度识别风险，输出 RISK_MODEL.md：

### 1. 业务风险
- 误操作导致的数据丢失（如：误删用户）
- 权限越权（如：普通用户看到管理功能）
- 业务规则冲突

### 2. 权限风险
- 不同角色看到不该看的数据
- 无权限用户通过URL直接访问
- 接口未校验权限

### 3. 数据风险
- 数据边界（空值、超长、特殊字符、SQL注入）
- 数据一致性（删除关联数据时的级联处理）
- 并发操作（两人同时编辑同一记录）

### 4. 接口风险
- 接口超时/失败时的前端表现
- 接口返回异常数据格式
- Token过期处理

### 5. UI/UX风险
- 不同分辨率下的布局问题
- 加载状态的用户体验
- 错误提示是否友好

### 6. 性能风险
- 大数据量下的表格渲染
- 频繁搜索时的请求频率

## 输出要求
- 每个风险标注等级（P0/P1/P2）
- P0 = 阻塞上线，P1 = 影响核心体验，P2 = 边缘场景
- 每个P0风险必须给出缓解措施
- 关联现有自动化用例覆盖率，标注未覆盖的高风险场景
```

## 检查清单

- [ ] 6 个维度全部覆盖（业务/权限/数据/接口/UI-UX/性能）
- [ ] 每个风险有唯一ID（如 RISK-ALARM-001）
- [ ] 每个风险标注了 P0/P1/P2 等级
- [ ] P0 风险全部有缓解措施
- [ ] P0 风险与 TEST_DESIGN 中的 P0 用例对应
- [ ] 数据风险覆盖边界值（空/超长/特殊字符/SQL注入）
- [ ] 权限风险覆盖"无权限用户通过URL直接访问"
- [ ] 高风险场景标注了现有自动化覆盖状态
