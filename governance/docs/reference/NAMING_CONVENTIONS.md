# Naming Conventions

## 总体规则
- 项目目录：使用英文短横线，如 web-automation
- 模块目录：与测试模块名保持一致，如 system-user
- 页面目录：使用页面语义短横线，如 user-list
- 文档文件名：使用全大写语义名，如 MODULE_CONTEXT.md

## 推荐文件名
- PROJECT_CONTEXT.md
- MODULE_INDEX.md
- MODULE_CONTEXT.md
- PAGE_CONTEXT.md
- RISK_MODEL.md
- TEST_DESIGN.md
- TEST_CASES.md
- TECH_ANALYSIS.md
- AUTO_STRATEGY.md
- PAGE_ELEMENT_POSITION.md
- TEST_SUMMARY.md

## 命名目标
- 可预测
- 可检索
- 可批量生成
- 可被 AI 稳定引用
