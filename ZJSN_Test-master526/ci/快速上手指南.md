# ZJSN 自动化测试 — Jenkins 快速上手指南（中文版）

> 适用场景：Windows 上用 Docker 运行 Jenkins，项目代码在本地磁盘
>
> 预计耗时：首次约 20 分钟

---

## 前置条件

- ✅ Docker Desktop 已安装并运行
- ✅ 项目代码在 `d:\Desktop\WorkStudy\ZJSN_Test-master526\`
- ✅ 能访问被测系统 `http://8.136.215.171:8081/`

---

## 一、停掉旧容器（如果之前跑过 Jenkins）

打开 PowerShell，执行：

```powershell
docker stop jenkins  2>$null
docker rm jenkins    2>$null
```

## 二、构建带测试环境的 Jenkins 镜像

```powershell
cd d:\Desktop\WorkStudy\ZJSN_Test-master526
docker build -t zjsn-jenkins:latest -f ci\jenkins.Dockerfile .
```

> ⏳ 这一步需要 5~10 分钟，因为要下载 Chrome 和 Python 依赖。看到 `✅ 环境就绪` 就说明成功了。

## 三、启动 Jenkins 容器

```powershell
docker run -d --name zjsn-jenkins `
  -p 8080:8080 `
  -p 50000:50000 `
  -v "d:\Desktop\WorkStudy\ZJSN_Test-master526:/var/jenkins_home/workspace/ZJSN_Test" `
  -v jenkins_home:/var/jenkins_home `
  --restart unless-stopped `
  zjsn-jenkins:latest
```

参数说明：

| 参数 | 作用 |
|------|------|
| `-p 8080:8080` | 浏览器访问 Jenkins 的端口 |
| `-p 50000:50000` | Agent 连接端口（备用） |
| `-v 项目路径:/var/.../ZJSN_Test` | 把代码挂载进容器，让 Jenkins 能读到 |
| `-v jenkins_home:/var/jenkins_home` | Jenkins 配置持久化（删容器不丢数据） |
| `--restart unless-stopped` | 电脑重启后自动启动 |

## 四、登录 Jenkins

### 4.1 获取初始密码

```powershell
Start-Sleep 30   # 等 Jenkins 启动
docker exec zjsn-jenkins cat /var/jenkins_home/secrets/initialAdminPassword
```

终端会打印一串类似 `a1b2c3d4e5f6...` 的密码，复制它。

### 4.2 完成初始化

1. 浏览器打开 `http://localhost:8080`
2. 粘贴初始密码 → **继续**
3. 选择 **安装推荐的插件**（等待几分钟）
4. 创建管理员用户（记好账号密码）
5. 实例配置保持默认 → **保存并完成**
6. 点 **开始使用 Jenkins**

## 五、安装额外插件

1. 左侧菜单 → **系统管理** → **插件管理** → **可选插件**
2. 搜索并勾选以下插件：

| 插件名 | 用途 |
|--------|------|
| **Allure** | 生成测试报告 |
| **Credentials Binding** | 安全存储密码 |
| **Email Extension** | 失败邮件通知 |

3. 点 **安装** → 勾选 **安装完成后重启 Jenkins**

## 六、配置 Allure 报告工具

1. **系统管理** → **全局工具配置**
2. 找到 **Allure Commandline** → 点 **新增 Allure Commandline**
3. 别名填 `Allure`，勾选 **自动安装**，版本选最新
4. 点 **保存**

## 七、添加项目凭据

1. **系统管理** → **凭据管理** → 点 **(global)** 下拉 → **添加凭据**
2. 添加以下 3 条 Secret text：

**凭据 ① — 测试环境地址：**

| 字段 | 值 |
|------|-----|
| 类型 | **Secret text** |
| ID | `zjsn-test-url` |
| Secret | `http://8.136.215.171:8081` |

**凭据 ② — 测试账号：**

| 字段 | 值 |
|------|-----|
| 类型 | **Secret text** |
| ID | `zjsn-test-username` |
| Secret | `admin` |

**凭据 ③ — 测试密码：**

| 字段 | 值 |
|------|-----|
| 类型 | **Secret text** |
| ID | `zjsn-test-password` |
| Secret | `你的实际测试密码` |

> ⚠️ ID 必须一字不差，Jenkinsfile 里按这个 ID 取值。

## 八、创建流水线任务

1. 回到仪表盘 → **新建任务**
2. 任务名称：`ZJSN_Test`
3. 选择 **流水线** → 确定
4. 往下滚动到 **流水线** 配置区域：

   | 配置项 | 值 |
   |--------|-----|
   | 定义 | **Pipeline script from SCM** |
   | SCM | **Git** |
   | 仓库 URL | `file:///var/jenkins_home/workspace/ZJSN_Test` |
   | 脚本路径 | `Jenkinsfile` |

5. 点 **保存**

## 九、首次构建

1. 进入 `ZJSN_Test` 任务页面
2. 点左侧 **参数化构建**
3. 弹出窗口确认参数：
   - `TARGET_ENV` → `test`
   - `RUN_REGRESSION` → 不勾选（首次先跑快的冒烟测试）
4. 点 **开始构建**
5. 页面下方会出现流水线进度条，点击构建号可看实时日志

构建阶段说明：

```
Setup    → 检查 Python + Chrome + 安装依赖
Lint     → 代码风格检查
Smoke    → 冒烟测试（约 5~15 分钟）
Report   → 生成 Allure 报告
```

## 十、查看测试报告

构建完成后，进入构建详情页：

- 左侧会出现 **Allure Report** 图标 → 点击查看完整报告
- 左侧 **构件存档** → 下载失败截图（如果有失败用例）

## 常用操作

### 日常跑测试

```
仪表盘 → ZJSN_Test → 参数化构建 → 开始构建
```

### 跑全量回归

```
仪表盘 → ZJSN_Test → 参数化构建 → 勾选 RUN_REGRESSION → 开始构建
```

### 只跑某个模块

```
参数化构建 → TEST_PATH 填: script/sales/test_contract.py → 开始构建
```

### 只跑某个标签

```
参数化构建 → TEST_MARKER 填: login → 开始构建
```

### 代码更新后

因为你用的是本地挂载（`file:///...`），Jenkins 直接读最新文件。改完代码后直接 **参数化构建** 即可，不需要 push。

## 故障排查

| 问题 | 解决方法 |
|------|---------|
| **Chrome 启动失败** | 确认 Jenkinsfile 中 `HEADLESS=true`；确认容器内存 ≥ 2GB |
| **Allure 报告为空** | 检查 pytest 有 `--alluredir` 参数；确认 Allure 工具已配置 |
| **凭据找不到** | 回到系统管理 → 凭据管理，确认 ID 一字不差 |
| **构建超时** | 超时默认 2 小时，如果不够在 Jenkinsfile 里改 `timeout` |
| **容器重启后 Jenkins 数据丢失** | 确认启动时有 `-v jenkins_home:/var/jenkins_home` |
| **端口 8080 被占用** | `docker ps` 查看是否有其他容器占用，改 `-p 9090:8080` |
